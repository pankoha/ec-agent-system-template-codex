# Gmail Amazon注文確定メール自動取り込み

`Codex依頼書 (1).docx` の内容を Google Apps Script 化したものです。Gmail に届く Amazon セラー通知メールから注文情報を抽出し、指定スプレッドシートへ追記します。

## 作成されるもの

- スプレッドシート名: `★注文確定商品リサーチ表★`
- 注文シート名: `注文確定商品リサーチ表`
- 確認用シート名: `確認用`
- Gmail 処理済みラベル: `処理済み`

注文シートの列構成:

| 列 | 見出し | 内容 |
| --- | --- | --- |
| A | 出荷予定日 | メール本文から抽出した出荷予定日 |
| B | 注文情報 | 注文番号、商品名、SKU、売上金を改行付きで集約 |
| C | 検索ワード | 通常商品は型番、DVD系商品は3行の検索ワード |

## セットアップ

1. 対象スプレッドシートを開く  
   `https://docs.google.com/spreadsheets/d/1fz_DnaigqwPvjQmRHyHagVFTR3tQRoLVHf46wqrjCSA/edit`
2. `拡張機能` → `Apps Script` を開く
3. [Code.compact.gs](./Code.compact.gs) の内容を Apps Script エディタの `Code.gs` に貼り付ける
4. `setupAmazonOrderImporterAndTrigger` を手動実行し、権限を承認する
5. スプレッドシートを再読み込みする

再読み込み後、メニューに `Amazon注文メール` が追加されます。`Amazon注文メールを取り込む` をクリックすると手動実行できます。

`setupAmazonOrderImporterAndTrigger` は、初期シート作成に加えて30分ごとの自動実行トリガーも作成します。手動でトリガー間隔を選びたい場合は、代わりに `setupAmazonOrderImporter` を実行してください。

取り込み後、`注文確定商品リサーチ表` はA列の `出荷予定日` を基準に昇順で並び替えられます。さらに、表示対象は `2026/06/01` 以降の出荷予定日に絞ります。既存行だけを並び替え・表示更新したい場合は、Apps Scriptで `showShipDatesFromJune2026` を実行してください。

既存行のB列 `売上金` をGmail本文の一番下にある `売上金：○○円` に直したい場合は、Apps Scriptで `refreshExistingSalesAmountsFromGmail` を実行してください。注文番号を使ってGmailを再検索し、B列の売上金行だけを更新します。処理時間対策として1回の実行で最大80注文まで確認するため、対象行が多い場合は同じ関数をもう一度実行してください。

既存行のB列/C列を、同一注文番号の複数商品を含めてGmail本文から作り直したい場合は、Apps Scriptで `refreshExistingOrderDetailsFromGmail` を実行してください。B列は `注文番号` の下に `【1】`、`【2】` の通番付きで商品名、SKU、売上金を再作成し、C列も同じ通番付きで検索ワードを再作成します。Gmailの `処理済み` ラベルを外す必要はありません。

`確認用` シートに `shipDate, items` で落ちた行をGmailから再処理したい場合は、Apps Scriptで `reprocessReviewRowsFromGmail` を実行してください。確認用D列の注文番号を使ってGmailを再検索し、取得できた注文を通常シートへ追加します。

## 自動実行トリガー

Apps Script エディタで、以下のいずれかを一度だけ実行してください。

- `installTimeDrivenTriggerEvery15Minutes`
- `installTimeDrivenTriggerEvery30Minutes`
- `installTimeDrivenTriggerEveryHour`

同じ `importAmazonOrderEmails` 用の既存トリガーは削除してから作り直すため、間隔を変更したいときも上記関数を再実行すれば大丈夫です。

## 抽出ルール

対象メール:

- 送信元: `seller-notification@amazon.co.jp`
- 件名に `注文確定` を含む
- Gmail ラベル `処理済み` が付いていない

抽出項目:

- 出荷予定日
- 注文番号
- 商品名
- SKU
- 売上金（メール本文内で一番下に記載されている `売上金：○○円` を採用）

重複防止:

- B列内に同じ注文番号がある場合は追記しません
- 1通のメール内に同じ注文番号で複数商品がある場合は、1行にまとめます
- B列の同じセル内に `【1】`、`【2】`、`【3】` のような通番を付け、各商品の商品名、SKU、売上金を記載します
- C列の検索ワードも同じ通番でまとめます
- 追記済みまたは重複スキップ済みのメールスレッドには `処理済み` ラベルを付けます
- 取り込み完了後、通常シートは出荷予定日の昇順で並び替えます
- `2026/06/01` より前の出荷予定日の行は、シート内の位置に関係なく非表示にします

取得失敗時:

- 通常シートには追記せず、`確認用` シートへ処理日時、受信日時、件名、取得できた情報、未取得項目、エラー内容を記録します

## 検索ワード生成

通常商品:

- 商品名から型番らしい英数字コードを抽出します
- 型番末尾のカラー表記 `-W/-K/-B/-S/-R/-N/-P/-H/-T/-C` は削除します
- 括弧内の `WS` などのカラー/仕様表記は削除します

DVD系商品:

商品名に以下の語が含まれる場合は DVD系として扱います。

- `レンタル落ち`
- `全○巻`
- `全巻セット`
- `DVD`
- `マーケットプレイスDVDセット商品`
- `Blu-ray`
- `ブルーレイ`

C列には以下の3行を出力します。

```text
作品名 全
作品名 巻数
作品名 レンタル
```

## 動作確認

Apps Script エディタで `testKeywordGeneration` を実行すると、依頼書に記載された検索ワード例が期待通り生成されるかを確認できます。

実メールの形式が想定と違う場合は、該当メールが `確認用` シートへ入ります。その内容を見ながら、`extractShipDate_` などの抽出パターンを追加してください。
