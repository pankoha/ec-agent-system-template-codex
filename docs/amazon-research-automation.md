# Amazon仕入れ候補リサーチ実装・検証記録

対象: `apps-script/amazon-order-importer`

仕様原本: [Codex実装指示書_改訂版.pdf](./Codex実装指示書_改訂版.pdf)

## 16節以降の実装対応

| 要件 | 実装 |
| --- | --- |
| D列の検索ワードを使用 | D列を改行分割し、各ワードを個別検索 |
| C列の売上金以内 | 価格と取得済み送料の合計で判定 |
| 送料不明 | 商品価格で仮判定し、結果へ `送料要確認` を付記 |
| ジャンク除外 | タイトル・状態・周辺テキストを禁止語で検査 |
| 既存URLを削除しない | 結果セルは追記のみ。削除処理を実装しない |
| URL重複防止 | 商品URLを正規化してセル内URLと比較 |
| 新規候補のみ追記 | 未登録の正規化URLだけを追記 |
| muza新品限定 | SKUに `muza` を含む行は新品状態のみ許可 |
| 表示中は継続 | E列の状態を終了条件にせず、表示行を毎時再処理 |
| Amazon | 新品および中古「可」まで |
| ヤフオク・メルカリ | 「傷や汚れあり」まで |
| ジモティ | ジャンクでなければ状態要確認を許可 |
| その他サイト | 楽天市場、駿河屋、オフモール、セカンドストリート |
| DVD・Blu-ray | 再生不可・欠品・巻数不足を除外 |
| 取得制限 | 処理を止めず、確認用シートへ検索URLを記録 |
| 表示開始日 | 2シートとも2026/06/30より前の行だけを非表示 |
| 手動編集保護 | 手入力・背景色・手動非表示行・削除列を変更／復元しない |

## 継続実行方式

`researchListedItemsHourly()` は【リサーチ管理表】で非表示になっていない行を収集します。`候補あり`や`候補なし`を含め、E列の状態は対象除外に使いません。

Apps Scriptの実行時間上限に備え、処理時間または行数の上限へ到達した場合はスクリプトプロパティにカーソルを保存します。次の1時間トリガーはカーソル位置から再開するため、先頭行だけが繰り返し処理されることを防ぎます。

## 検証

`tests/amazon-order-importer-research.test.js` で以下を自動検証します。

- 価格と送料の上限判定
- 送料不明表示
- ジャンク・動作未確認除外
- サイト別の状態条件
- muza中古除外／新品許可
- DVD全巻判定
- 型番一致
- URL正規化と重複防止
- 表示行のみの選択
- カーソル循環
- 既存URLおよび手動入力の保持
