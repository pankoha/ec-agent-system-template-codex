# Security Policy / セキュリティポリシー

## 機密情報の取り扱い（最優先）

本テンプレートを使用する際、**以下を絶対に守ってください**。違反すると APIキー・認証情報がインターネットに公開され、不正利用の被害を受けます。

### やること（MUST）

1. **`.env` ファイルを絶対に git commit しない**
   - 本リポジトリの `.gitignore` に既に `.env` 等を除外設定済み
   - 自分で追加したファイルも必ず `.gitignore` に含める
   - コミット前に `git status` を確認する癖をつける

2. **API キー・トークンをコード内にハードコードしない**
   - 必ず環境変数経由で参照する（`.env` + `dotenv` など）
   - `.env.example` にはキー名のみ記載し、値は `YOUR_API_KEY_HERE` 等のプレースホルダーに

3. **公開前にスキャンする**
   ```bash
   # gitleaks で機密情報の漏洩チェック（初回のみインストール）
   brew install gitleaks                      # macOS
   gitleaks directory . -v                    # 作業ディレクトリスキャン
   gitleaks git --log-opts="--all" -v         # 全履歴スキャン
   ```

4. **pre-commit hook を設定する**
   - `./setup/install-hooks.sh` を実行すると自動で `.git/hooks/pre-commit` に gitleaks スキャンを仕込みます
   - 以降、commit 時に機密情報が含まれていると自動でブロックされます

### やってはいけないこと（NEVER）

- ❌ `git add -A` / `git add .` を確認せずに実行する
- ❌ `.env` をエディタで開いたまま画面共有する
- ❌ API キーを Slack / Chatwork / メール本文にコピペする
- ❌ `git push --force` で履歴を書き換える（漏洩した鍵が残り続ける）
- ❌ Claude Code の deny 設定だけを信じて `.gitignore` を省略する（deny はローカル制限のみ、GitHub 公開は止められない）

---

## 多層防御の優先順位

本テンプレートでは、機密情報の漏洩対策を **優先度の高い順に以下の4層** で守る設計を採用しています。

| 優先度 | 層 | 目的 | 外れたときの被害 |
|---|---|---|---|
| 1 | `.gitignore`（最優先・必須） | そもそも git に乗せない | ネット公開（致命的） |
| 2 | `~/.gitignore_global` | 新規リポジトリでも効く保険 | プロジェクト作成時の漏れ |
| 3 | Claude Code の deny リスト（`~/.claude/settings.json`） | AI が読まない/送信しない | AI が誤って外部送信 |
| 4 | pre-commit hook（gitleaks） | commit 時の自動スキャン | 手作業の抜け |

**重要**: 層 1 が最優先。層 3 だけに頼るのは危険です。deny はローカル AI の読み取りを止めるだけで、GitHub に push された `.env` は人間・クローラー・他の AI に読まれます。

---

## もし漏洩してしまったら

1. **直ちに該当キーを revoke（再発行）** する
   - 履歴から削除するより**鍵の無効化が優先**
2. 新しい鍵を発行し `.env` に置き換える
3. 履歴に残っている場合は `git filter-repo` で過去コミットから削除（完全消去は不可能と理解した上で）
4. 漏洩した期間の AWS / GCP / API 利用状況を確認し、不正利用がないかチェック
