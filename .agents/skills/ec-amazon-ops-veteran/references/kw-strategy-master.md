# KW Strategy Master（KW戦略マスター）

ブランド分析を中心としたKW戦略の完全マスターガイド。

## A. ブランド分析の権限問題（重要）

```
ブランド分析が出てこない
   ↓
原因：アカウント単位の権限付与のみ
   ↓
対処：グローバル全体の権限を付与してもらう
```

→ 多くの人がここでつまずく・要注意

---

## B. SEOヒット項目と優先順位（決定版）

```
1位：商品名（最重要）
2位：箇条書き
3位：A+コンテンツのテキスト
4位：裏キーワード（バックエンド）

参考：レビュー（アンコントローラブル・除外）
```

### 覚え方（2軸）
- **お客さんに見える情報の方が優先**
- **PC表示の上から順に大切**

---

## C. 各項目の文字数制限（実務知識）

| 項目 | 制限 |
|---|---|
| 商品名 | 50文字以内推奨（アパレル80） |
| 箇条書き | 1000文字以内推奨（カテゴリ拡大あり） |
| A+テキスト | 制限なし |
| 裏キーワード | **500バイト=約165文字（厳格）** |

### 商品名の特別注意点
- **完全一致必須**：単語間に半角スペースなければ別単語認識
- **変更すると順位ガクッと落ちる** → 商品登録時に1発で決める

### 裏キーワードの厳格性
- **165文字超えるとインデックスから外れる**
- 他項目と違い超過するとSEO効果失われる
- 超過時はテクサポに原因確認→修正

## E. KW戦略の見極めツール

### Aloha 10等のChrome拡張
- 複数KWを一発で検索順位確認
- 戦略KW3つ程度を週1チェック推奨

### 戦略KW選定例
- 主力カテゴリの大ボリュームKW（例：カテゴリ総称名）
- 関連カテゴリのミドルKW（例：派生カテゴリ名）
- 用途KW（例：用途別表現）
- → 3つ程度を「絶対モニタリング」と決める

## F. 順位低下時の改善フロー

```
順位低下発覚
   ↓
Step 1: CTRL+Fでページ内検索
        商品名・箇条書き・A+にKW入ってるか
   ↓
Step 2: 入っていなければ追加
   ↓
Step 3: 入っているのに順位低い → 複数入れる
        【独自ノウハウ】箇条書きには1個より複数入れた方が効果的
   ↓
Step 4: A+にも3回程度繰り返し挿入
```

### 独自ノウハウ：「複数挿入」
- 公にされていないテクニック
- 複数のテストで効果確認済み
- 重要KWを箇条書き3回、A+3回挿入

## G. ブランド分析の数値判断基準（最重要）

### ランキング解釈
```
3桁(1-999)    : 超強い
4桁(1000-9999): かなり強い
1-3万         : そこそこ
3-5万         : 角落ち
5-10万        : ちょい角落ち（ある程度ボリューム）
10万超(6桁)   : ついで対策レベル
```

### マーケット内の相対評価が本質
```
大きなマーケット（家電・家具等）：
- メインKW 4000位台 → でかい
- 用途KW 1000位台 → バカでかい

小さなマーケット（ニッチサプリ等）：
- メインKWですら6桁ランク
- それでも狙う（マーケットがちっちゃいから）

→ 絶対ランクで切らない・マーケット内相対で判断
```

## H. 検索ボリュームと競合度のトレードオフ（戦略的判断）

### でかいKWの罠
| 項目 | でかいKW | 中規模KW |
|---|---|---|
| マーケット | 大 | 中 |
| 競合数 | 多 | 中 |
| CPC | 高騰 | 適正 |
| 上位表示難易度 | 高 | 中 |

### 攻める優先度の決め方
```
最大KW
   ↓ 商標問題＋競合最強で攻めにくい
中規模KW
   ↓ ←攻める優先度高い
小規模KW
   ↓ 補助的に
```

→ **「ちょい落ちるが超でかい」が狙い目**

## I. 期間設定の重要原則

```
週 < 月 < 四半期
   ↓ データボリューム
   ↓ 信頼度
基本は四半期で見る
```

- たまたま先週増えただけ等の誤判定回避

## J. KWの軍制（戦略的配置）

| 軍 | 配置先 | KW例 |
|---|---|---|
| **1軍** | 商品名 | メイン3KW程度 |
| **2軍** | 箇条書き | 修飾語・特性（折りたたみ・正音・コンパクト） |
| **3軍** | A+ | 補助訴求・ベネフィット説明 |
| **4軍** | 裏キーワード | 修飾語・関連語・ロングテール |

### 1軍候補の絞り込み
- マーケット内相対で大きさ確認
- 競合度確認
- 商標問題確認（他社が商標取っているKWは使えない場合あり）
- 商品特性とのマッチ確認
- → 3つ程度に厳選

## K. Amazonサジェスト活用

### 特性
- 1〜31分単位で順番が変わる（リアルタイム）
- 短期トレンド・テレビ紹介後の流行KW発見に有効
- 中長期で見ればボリューム少ない場合も

### 使い分け
- **本質：ブランド分析**（中長期データ）
- **補助：サジェスト**（トレンド・短期発見）

## L. 検索クエリパフォーマンス（上級機能）

### 概要
- 2022年末リリース
- 自社商品のみ閲覧可能
- ブランド分析の別ツール

### 表示項目
| 項目 | 意味 |
|---|---|
| 検索KW | 自社商品が表示されたKW |
| クエリスコア | 検索ボリューム×CTR×購入数の総合ランキング |
| 検索ボリューム | 数字大きい=ボリューム大 |
| 購入数 | ASIN単位で何個買われたか |

### 重要：相性の見抜き方
```
ケースA：検索ボリューム少・購入数多
  → 相性◎（隠れた金脈KW）

ケースB：検索ボリューム多・購入数少
  → 相性×（狙う必要なし）
```

### 活用シーン
1. **バージョンアップ商品開発時**：既存売れ筋のKWベース分析
2. **広告抜け漏れ発見**：自然検索で売れてるが広告未出向KWを発見
3. **ターゲ追加**：意外なKWで実は売れていた発見

### 事例パターン
```
あるカテゴリ商品：
- KW A：検索ボリューム少・購入3個 → 相性◎ → 広告追加
- KW B：検索ボリューム多・購入1個 → 相性△ → 維持
- KW C：自然検索で売れてるが広告未出 → 即広告追加
```

## M. 競合分析の前提

### 「皆さん意外と競合をマジで見ない」
- 競合を見るだけで差別化できる
- 同カテゴリのトップ売れ筋商品名を解剖
- 使われているKW・スペース有無・順序を確認

### 商標問題への注意
- 一部のKWは他社が商標取得している場合あり
- 自社商品に使えない可能性
- 必ず確認

```
【リサーチフェーズ】

【モニタリングフェーズ】
```

## O. 商品名作成 4つのルール

### ルール1：文字数制限
- **アパレル以外：全角50文字以内推奨**

### ルール2：重要KWを前に
- 検索ボリューム多いKWを前に配置

### ルール3：完全一致（最致命的注意点）
```
NG: 単語1単語2（スペースなし）
   ↓ → 「単語1単語2」で1単語認識

OK: 単語1 単語2（半角スペース）
   ↓ → 「単語1」「単語2」両方認識
```

### ルール4：順序
- **ブランド名 → 商品名 → 用途/特性 → 使用色・サイズ**

## P. 薬機法注意（カテゴリ依存）

### 使えないKWパターン
| カテゴリ | NG例 |
|---|---|
| 効果系 | 効果あります・改善できます |
| 症状名 | ニキビ・湿疹等（カテゴリ依存） |
| 人体部位 | 肌・肘・関節（カテゴリ依存） |

### カテゴリによる例外
- 化粧品：「肌」はギリギリ使えることもある
- サプリ：「肌」も使えない
- → **商品カテゴリで毎回確認必要**
