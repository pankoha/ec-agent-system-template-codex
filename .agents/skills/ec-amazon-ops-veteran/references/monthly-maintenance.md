# Monthly Maintenance（月次メンテナンス 理論+実演）

月2回ペースで最大インパクトを出す月次メンテナンスフレームワーク。
理論パートと現場実演パートを統合。

---

## 基本ルール

### メンテ頻度の黄金式

```
通常運用      : 月2回（2週間に1回）
初期設定直後  : ちょくちょく確認（1週間に2-3回）
SD配信開始直後: ちょくちょく確認（暴走しやすい）
実際の操作    : 多くて週1か月2回
```

#### 「月2回」の根拠
- 1週間だとデータ溜まりきってない
- 2週間でようやく「本当に良かったか」判断可能
- データの統計的有意性を待つ判断

---

## 全体像把握ステップ（毎回）

### Step 1: 先月の数字確認
- 期間を「先月」に切り替え
- 広告費・売上・ROAS・ACOS を全体で見る
- 前月比較で傾向把握

### Step 2: 全体ACOSの計算
```
事例：
広告費 86万 / アカウント全体売上 220万 = 全体ACOS 40%

評価基準：
20%以下  : 効率良い
20-30%   : 普通
30-40%   : やや高い
40%超    : 高い（攻めフェーズなら健全な場合もある）
```

### Step 3: メンテ方向性の決定
- クライアントコミュニケーション踏まえて「やや強める」or「弱める」決定
- クライアント戦略との整合性確認

---

## 優先順位付け：広告費降順（鉄則）

### 並び順
```
キャンペーン一覧 → 広告費の降順ソート
```

### 理由
- 広告費いっぱい使ってるキャンペーン = 修正インパクト最大
- 全部こまめに触ると切りがない
- **上から順に対応＝最大インパクト最短実現**

---

## キャンペーン階層での判断

### 全体ROAS基準で比較
```
全体平均ROAS = 1.07
  ↓
キャンペーンROAS 0.82 → やや悪い → 下げる
キャンペーンROAS 0.9  → やや低い → CPC下げ基本
キャンペーンROAS 1.5  → 良い → CPC上げ検討
キャンペーンROAS 2.0以上 → 神KW候補 → 大胆攻め
```

---

## ターゲ階層に降りる（中身判断）

### 階層構造
```
キャンペーン
  ↓
グループ（KWターゲ／ASINターゲ／カテゴリターゲ）
  ↓
個別ターゲ（KW1個・ASIN1個・カテゴリ1個）
  ↓ ここでCPC調整
```

### グループ別の成績差
- マニュアルターゲでも「KWターゲグループ」と「ASINターゲグループ」で成績違う
- 必ず1個1個のターゲを見に行く

---

## CPC調整の判断ロジック

### 上げ判断のサイン
| 指標 | 判定 |
|---|---|
| 推奨入札 < 現状CPC | 既に攻め状態（さらに上げる余地） |
| ROAS良好（平均の1.5倍以上） | 上げて購入数増狙い |
| 仮説 | 「ACOS下がるけど購入数増えるかも」 |

### 下げ判断のサイン
| 指標 | 判定 |
|---|---|
| CPC高 + ROAS低 | 効率悪化中 |
| 広告費 > 売上 | 完全な赤字状態 |
| 平均CPC1000円超 + ROAS0.3未満 | 即座に下げ |

---

## 調整幅の感覚値

```
通常調整    : 20-30円幅
大胆調整    : 50-60円幅
グッと下げ  : 3/4または半分
グッと上げ  : 50円幅程度

ロジックで詰めすぎない・感覚でOK
```

→ 「これ1個1個ロジックで基づいてやってると本当切りない」

---

## 神KW・神ターゲの判定基準

### 全体平均ROASの倍以上＝神
```
全体平均ROAS 1.0の場合：
- ROAS 3.0以上 → 神KW（大胆攻め）
- ROAS 2.0-3.0 → 強KW（50円幅で上げ）
- ROAS 1.5-2.0 → 良KW（20-30円幅で上げ）
- ROAS 1.0-1.5 → 普通（維持）
- ROAS 0.7-1.0 → 悪い（20-30円幅で下げ）
- ROAS 0.7未満 → 即下げ（半分など）
```

→ **平均の2-3倍ROASのターゲは攻めるべき優先案件**

---

## CPC変更後の即時確認テクニック

### Amazon検索で順位確認
- CPC上げ後 **30秒〜1分** で実際に順位反映
- 「中段→上位」とリアルタイムで動く

### 攻めの応用
- 200円→250円とさらに強気
- 神ASIN/KWは「スポンサー枠への露出」を狙って大胆上げ

---

## クライアント戦略との整合性（中級編・最重要）

### 重要：単純な「ROAS改善」が常に正解ではない

```
ケース：CPC 210→150に下げた
  ↓
ROAS：良くなる ✓
注文数：減る可能性 ✗

クライアントが「注文数優先・売上トップライン狙い」なら
→ 下げない判断もアリ
```

### 戦略タイプ別の動き方

#### 攻め型クライアント
- 注文数増やしてトップライン狙い
- → 基本「攻め」モード・CPC上げ基本
- ROASやや悪化しても許容

#### 守り型クライアント
- 利益率重視・赤字許容できない
- → 基本「守り」モード・CPC下げ基本
- 注文数減少も許容

#### バランス型クライアント
- フェーズに応じて使い分け
- 商品別に攻め/守りを分離

→ **クライアント前提を必ず確認してから判断**

## 実演フロー（事例ベース）

### 月初の作業フロー
```
1. 前月数字確認（広告費・売上・ROAS・ACOS）
   ↓
2. 全体ACOS計算（広告費/全体売上）
   ↓
3. クライアント戦略確認（攻め/守り）
   ↓
4. キャンペーン一覧 → 広告費降順ソート
   ↓
5. 上から順にキャンペーン開く
   ↓
6. ROAS判定 → 全体平均との比較
   ↓
7. グループ階層に降りる
   ↓
8. ターゲ階層まで降りる
   ↓
9. 1個1個のCPC判断
   ・神KW（平均ROAS倍以上）→ 大胆上げ
   ・良KW → 微上げ
   ・悪KW → CPC下げ（止めない）
   ↓
10. 主要KWはAmazon検索で順位確認
   ↓
11. 全キャンペーン繰り返し
```

### 月末の作業フロー
```
1. 今月の数字集計
   ↓
2. KPI達成状況確認
   ↓
3. 翌月の方針決定
   ↓
4. クライアントへのレポート作成
   ↓
5. 「こうしました」を必ず記録
```

---

## メンテ品質の前提：データ準備

### 自動入力シートの活用
- 手動でビジレポ整形は持続不可能
- 月次レポート効率化ツールが必須
- データ準備時間1/3にして判断時間に集中

### 並び順の使い分け
- 月末レポート：売上順
- 中間レポート：固定ASIN比較（経時変化）

---

## 「素人運用」を避ける禁忌

```
❌ 「悪かったら止める」短絡判断
   → CPC下げて維持を試す（止めるのは最終手段）

❌ 大きなKWを簡単に止める
   → 全体への影響大きい・CPC調整で対応

❌ オートターゲに広告費集中放置
   → マニュアル展開でコントロール領域広げる

❌ ASIN「拡張」ターゲ使用
   → 完全一致に切り替え

❌ クライアント前提なき提案
   → 攻め型/守り型を必ず確認

❌ データ見ずに感覚で判断
   → 必ずビジレポと広告レポートを並べて見る
```
