# Account Handover Diagnosis（新規アカウント引継ぎ診断）

新規受託・引継ぎ時のアカウント診断フレームワーク。
過去多数の引継ぎ実例から抽出した「素人運用サイン」の発見手順。

---

## 診断の8セクション構成

```
A. ビジネス全体把握
B. ブランド戦略推測
C. Amazon基本機能活用度
D. 商品ページ品質
E. SEO・KW
F. 広告構造
G. ヒアリング項目化
H. 「素人運用サイン」スコアリング
```

---

## A. ビジネス全体把握

### A-1. 売上推移2年分の異常検知

```
過去24ヶ月の月別売上を確認
   ↓
Amazon定常ピークを除外して異常月を特定
   ・7月（プライムデー）
   ・11月（ブラックフライデー）
   ↓
これら以外で急上昇/急下降した月を特定
   ↓
売上ベースが変わったポイントをマーキング
   ↓
ヒアリング項目として記録（推測しない）
```

#### 想定される異常月の理由
- 外部メディア露出
- 競合の撤退/参入
- 大型施策実行
- SNS拡散
- 在庫切れ
- 価格改定
- カテゴリ変更

### A-2. 商品理解（カテゴリ・ベネフィット）
- 商品名・パッケージ・公式LP・レビューを横断確認
- 「お客さんはなぜこの商品を買うか」を3つ以上言語化
- 知らない商品でも理解しないと診断不能

### A-3. シーズナリティ判定
- 仮説立案 → データ検証の2ステップ
- 例：体内塩分関連商品 → 夏に汗で塩分濃度下がるから夏売れる仮説
- 過去2年データで仮説検証

### A-4. バリエーション設定状況
- 同じ商品ラインで別ASIN（色違い・香り違い）が並んでいないか
- バリエーション組まれていない → レビュー分散・SEO的不利
- なぜ組んでいないかをヒアリング項目に必ず追加

---

## B. ブランド戦略推測

### B-1. 公式LP/Amazon価格比較
```
Google検索でブランド名
   ↓
公式LP/Google広告LPの価格確認
   ↓
Amazon価格と比較
```

#### パターン1：公式重視型
- 公式の方が安い
- 公式の定期購入顧客がAmazonに流出するのを防ぐ
- Amazonセールは抑制方針が推測できる

#### パターン2：Amazon重視型
- Amazonの方が安い・限定特典
- Amazon主軸の販売戦略
- セール積極参加

#### パターン3：併売型
- 価格同一
- チャネル中立

→ どれを採用しているかでAmazon運用方針が変わる

### B-2. セール参加履歴（Chrome拡張で確認）
- Keepa等で過去の価格推移グラフを確認
- 特選タイムセール参加履歴
- プライム会員限定セール参加履歴
- 全く未実施なら「公式重視のためAmazonセール抑制」推測

### B-3. Google広告/SEO状況
- 公式LP のスポンサー出向有無
- メインKWでの自然検索順位
- → Amazon広告と外部広告の相関を把握

## C. Amazon基本機能活用度

| 機能 | 役割 | 未活用の影響 |
|---|---|---|
| FBA入庫 | プライム表示・配送リードタイム | プライムなし→CVR大幅低下 |
| Vine | 公式レビュー収集 | レビュー獲得速度低下 |
| クーポン | CVR向上施策 | 即効性のあるCVR改善機会喪失 |
| 定期便（Subscribe & Save） | リピート獲得 | LTV向上機会喪失 |
| ブランドストア | 公式機能 | 統合的なブランド訴求機会喪失 |
| ブランドストーリー | 商品ページ内訴求 | プレミアムA+の前提条件不充足 |
| A+コンテンツ | 商品ページ拡張 | CVR向上機会喪失 |

### FBA入庫の確認方法（最優先）
```
商品ページ → 出荷元欄

セラー名表示 → FBA未利用 ✗
Amazon表示  → FBA利用 ✓
```

→ FBA未利用は **「提案するだけで超売上が上がる」第一の改善ポイント**

## D. 商品ページ品質

### D-1. サブ画像チェック
- メリット・ベネフィットが伝わるか
- 「使ってどうなるの？」が分かるか
- 画像比率の統一感（正方形/長方形混在は NG）
- スマホ表示時の見え方

### D-2. A+コンテンツチェック
- サブ画像と同じ画像の使い回し有無
- 独自コンテンツの量
- クオリティ評価
- メインKW挿入回数（複数回が理想）

### D-3. ブランドストーリーチェック
- 設定有無
- 設定済みなら内容クオリティ

### D-4. ブランドストアチェック
- 設定有無
- 画像クオリティ
- 商品ページへの誘導設計

---

## E. SEO・KW

### E-1. ブランド分析「上位の検索キーワード」
- 月別売上ランキングKWを長期間で確認
- メインKWの売上規模把握
- 大ボリュームKWを特定

### E-2. 商品名・箇条書きチェック
- メインKWが商品名に入っているか
- 箇条書きにメインKWが入っているか
- 過剰修飾語の整理状況

## F. 広告構造

### F-1. 命名規則の有無（即診断可能）
```
キャンペーン名規則性ゼロ
   ↓
集計不能・属性別判断不能
   ↓
即修正対象（運用品質の根幹）
```

→ 動画7・14で言及された通り、命名規則の崩壊はデータドリブン運用の前提を破壊する

### F-2. 広告費・売上推移
- 全期間で広告費と売上の推移
- ROASの変動を確認
- 急変ポイント特定

#### 典型的な「業者投入後の崩壊」パターン
```
ある月から広告費が爆発的に増加
   ↓
ROASが継続的に下落
   ↓
売上は伸びず、広告費だけ増加
```

→ 外部業者の暴走的運用が疑われる

### F-3. キャンペーン構造
- 商品別の広告投資バランス
- 主力商品に集中、他商品ゼロは NG
- 全商品にちゃんと広告できているか

### F-4. オート/マニュアル配分
- オートターゲに広告費集中している場合は素人運用サイン
- マニュアル展開できていない

### F-5. ASIN拡張使用有無（NG確認）
- 拡張は基本NG
- 類事商品に無制限拡散
- 完全一致のみが原則

### F-6. SP/SB/SD全活用度
- 3種揃って運用されているか
- SD暴走→完全停止していないか

### F-7. 「悪かったら止める」事故有無
- 大きなKWを「悪いから止めた」履歴
- CPC下げて維持の選択肢を取れていない

### F-8. 未承認広告の放置有無
- 「未承認」のまま予算組まれているキャンペーン
- 静かな機会損失

## G. ヒアリング項目化

### 推測で進めない・確認項目は分離

```markdown
## ヒアリング項目（クライアント確認待ち）

1. 売上ベース変動の理由
   - YYYY年MM月から売上ベースが変わった理由
   - 何か施策・出来事があったか

2. バリエーション組まない理由
   - 同シリーズで別ASIN展開している意図
   - 統合の検討状況

3. セール抑制の理由
   - Keepa上ではセール履歴がない
   - 公式重視戦略か・他の理由か

4. 業者運用履歴・現状
   - 過去に外部業者で運用していた期間
   - 引継ぎ時の状態

5. ブランドの販売チャネル戦略
   - Amazon／公式LP／楽天等の優先順位
   - チャネル間カニバリ意識
```

## H. 「素人運用サイン」スコアリング

### 5サインチェック

```
□ 1. キャンペーン命名規則ゼロ
□ 2. オートターゲに広告費集中
□ 3. ASIN拡張ターゲ使用
□ 4. 大きなKWを「悪いから止めた」
□ 5. SD広告暴走→完全停止
```

### スコア解釈
- 0-1個：熟練運用・改善余地小
- 2-3個：中級運用・改善余地中
- 4-5個：素人運用・改善余地大（ガラ空き＝伸び代）

### Quick Win 候補抽出
診断後、即実行できる3つの改善ポイントを抽出：
1. FBA未利用なら → FBA入庫提案（CVR劇的改善）
2. 命名規則ゼロなら → キャンペーン再構築（集計可能化）
3. ASIN拡張使用なら → 完全一致化（広告費効率化）
