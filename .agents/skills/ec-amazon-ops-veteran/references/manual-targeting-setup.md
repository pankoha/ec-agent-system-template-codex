# SP Manual Targeting Setup（マニュアル広告 完全設定マニュアル）

スポンサープロダクトのマニュアル広告（KW・ASIN・カテゴリ）の完全設定手順。

---

## 大前提：データソースの優先順位

```
データの取り方
└─ オートターゲがある → 検索用語（過去65日）から抽出 ★最優先
└─ オートデータがない → Chrome拡張でASIN抽出
                       ＋ ブランド分析でKW抽出
                       ＋ Amazonサジェストでロングテール抽出
```

---

## A. マニュアルKWターゲ 設定フロー

### 1. オートターゲから検索用語を抽出

```
オートターゲのグループ
   ↓
検索用語タブ
   ↓
過去65日に設定（最大期間）
   ↓
注文の降順でソート
   ↓
オートの平均ACOSを基準把握（例：136.9%）
```

### 2. KWの仕分け
- 注文数のあるKWのみ抽出（CV0は捨てる）
- KWとASINを別グループに振り分け
- 「KWターゲグループ」「ASINターゲグループ」を別作成

### 3. 効率悪いKWの扱い方（重要）
- **効率悪いから切るのではない**
- CPC下げて入れる
- 例：130円で悪い → 60-50円なら効率良くなる可能性

### 4. 一致タイプ設定（最重要警告）

#### デフォルト3つの一致タイプ
- 部分一致（broad）
- フレーズ一致（phrase）
- 完全一致（exact）

#### 設定ルール
```
「部分一致」を必ず外す
   ↓
理由：「キー中の1個でも一致したらターゲ」＝めちゃくちゃ広い
   ↓
→ フレーズ一致＋完全一致のみで設定
```

### 5. 推奨入札額の罠
- Amazon推奨は元のCPCより大幅に高いことが多い
- 例：93円で61%ACOSなのに、推奨300円が出ることも
- → **推奨を信用せず、元のCPCベースで設定**

### 6. 一括設定の現実解
- 個別に1つずつCPC設定は時間がかかりすぎる
- 「カスタム入札」で全部90円などにまとめて入れる
- 後から最適化

---

## B. マニュアルASINターゲ 設定フロー

### オートにデータがある場合
- 過去65日の検索用語からASIN抽出
- 100円程度で一括設定

### オートにデータがない場合：Chrome拡張活用
- 狙いたいKWでAmazon検索
- Google拡張機能（ASIN一括抽出ツール）でASINリスト取得
- 上位100個程度を採用（200位以下は無関係になりがち）
- 関係なさそうなものを削除

### 完全一致 vs 拡張（最重要警告）

| 設定 | 動作 | 採用基準 |
|---|---|---|
| **完全一致** | 指定ASINのみ | **基本これ** |
| 拡張 | そのASINと似たもの全部に拡散 | **基本使わない** |

#### 拡張がNGな理由
- 118個 × 類事商品 = めちゃくちゃ広くなる
- 効率悪化が確定的
- コントロール不能
- 意図的な拡散したい時のみ使用

---

## C. カテゴリーターゲ 設定フロー

### 概念
- 特定カテゴリ全体にバッとかける
- 「**オートターゲのASIN絞り込み版**」
- 実際どこにかかるかは回してみないと分からない

### 絞り込み機能（カテゴリーターゲ最大の武器）

#### レビュー評価で絞る
- **星3以下のものだけにかける**
- → 弱い競合を狙って奪う戦略

#### 価格帯で絞る
- **自社と同程度の価格帯だけ**
- 例：自社4,950円 → 3,000円以上に絞る
- 高単価商品同士の方がCVR高い（同価格帯ユーザーへの最適化）

### CPC設定
- カテゴリーターゲは幅広くかけるため自然とCPC高くなりがち
- → **通常より低めに設定**（KWターゲ90円なら、カテゴリー50円）

---

## D. 概念的金言：広告効率の本質

> **「狙ってる人1人に集中」vs「全員に声かけ」**
>
> - ピンポイント狙い = CPC低・CVR高・労力小・狙い通り
> - 幅広く狙い = CPC高・CVR低・労力大・偶然の出会い

→ カテゴリーターゲ・オートターゲは「全員声かけ」型なので、コスパ低めが当たり前。CPCを下げて入れる。

---

## E. キャンペーン構造の鉄則

```
SP_マニュアル_商品名（1キャンペーン）
  ├─ グループ: キーワードターゲ
  ├─ グループ: ASINターゲ（完全一致）
  └─ グループ: カテゴリーターゲ（絞り込み付き）
```

### キャンペーン乱立はメンテ困難
- グループで分けて管理
- 命名規則必須（プレフィックスを必ず入れる）

---

## F. 入札戦略・予算設定

| 項目 | 設定 |
|---|---|
| 入札戦略 | **「アップとダウン」禁・「ダウンのみ」必須** |
| 予算 | 3,000円スタート → 後から効率に応じて増減 |
| 「3,000円設定」の意味 | 最大3,000円まで使う・必ず使うわけではない |

### 「アップとダウン」を使うべき例外フェーズ
- ROAS悪くてもいいから売上トップライン上げたい
- 検索順位上げに行きたい立ち上げ期
- 「攻める」と決めた商品のみ

→ デフォルト「ダウンのみ」が正解

---

## G. 設定後のオペレーション

### 初期フォロー（1週間に2-3回）
- カスタム入札で一括設定したCPCの調整
- 効率悪い個別ターゲのCPC下げ
- 神KW候補のCPC上げ

### 通常運用（月2回）
- 月次メンテナンスに統合
- 広告費降順で対応

### KW昇格パイプライン
```
オート検索用語
   ↓ 注文ありKW抽出
フレーズ一致＋完全一致
   ↓ ROAS良いKW
完全一致（CPC上げて攻め）
```

---

## H. ASIN拡張使用は即修正対象

### 引継ぎ診断でASIN拡張使用を発見したら
```
即実行：
1. 拡張使用しているターゲをリストアップ
2. 完全一致に変更
3. 新しいキャンペーンに切り出し
4. 1週間モニタリング
5. 効率改善を確認

予想される効果：
- 広告費削減（無駄な拡散停止）
- ROAS改善
- コントロール可能性回復
```

## チェックリスト

```
□ オートターゲのデータを過去65日で抽出済みか
□ 注文ありKWのみ抽出しているか
□ KWとASINを別グループに分けているか
□ 部分一致を外しているか（フレーズ+完全のみ）
□ Amazon推奨入札を盲信していないか
□ ASIN「拡張」を使っていないか（完全一致のみ）
□ カテゴリターゲで絞り込み（レビュー/価格）使っているか
□ カテゴリターゲのCPCを通常より低めに設定しているか
□ 入札戦略を「ダウンのみ」にしているか
□ 1キャンペーン内にKW/ASIN/カテゴリのグループ分離があるか
□ キャンペーン名にプレフィックス（SP_マニュアル_商品名）入れているか
```
