# TREND INTELLIGENCE 運用ルーティン 詳細

## 日次ルーティン
```
日次:
├── ニュースフィード監視（US/APAC/Global）
├── GitHub Trending チェック
└── Critical Alert の即時配信
```

### 日次監視チェックリスト
```
□ 主要テックニュースサイトのチェック
□ GitHub Trending (All languages / Python / TypeScript)
□ HackerNews Top Stories
□ Product Hunt 新着プロダクト
□ 競合の新発表・プレスリリース
□ AI関連の新論文・発表
□ EC関連の規制変更ニュース
```

## 週次ルーティン
```
週次:
```

## 月次ルーティン
```
月次:
├── 月次トレンドレポートの作成
├── Technology Radar の更新
├── Best Practice Library の更新
└── 四半期トレンド予測の更新
```

### Technology Radar 更新プロセス
```
カテゴリ:
├── Adopt（採用推奨）: 実績あり、即導入可能
├── Trial（試行推奨）: 有望、PoC段階
├── Assess（評価中）: 注目すべき、調査段階
└── Hold（保留）: 時期尚早、またはリスクあり

更新手順:
1. 前月のRadar項目をレビュー
2. 新規トレンドの追加判断
3. 既存項目のステータス変更
4. 各Agent担当者へ通知
```
