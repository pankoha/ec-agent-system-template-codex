# 公式LINE構築 詳細ガイド（Amazon物販3ステップメソッド）

## 公式LINEの役割

```
Amazon自社ブランドにおけるLINEの位置づけ:
├── プリローンチの事前集客チャネル
├── 新商品リリース時の告知チャネル
├── 大型セールの事前告知
├── 購入者とのコミュニケーション（ファン化）
├── レビュー依頼の導線
├── ★ Amazonでは顧客データが取れない → LINEで顧客関係を構築
└── ★ 「Amazonの弱点（顧客接点の少なさ）をLINEで補完する」
```

## ツール選定

```
LINE公式アカウント + 拡張ツール:

LINE公式アカウント（基本）:
├── 無料プラン: 月200通まで無料
├── ライトプラン: 月5,000通 ¥5,000/月
├── スタンダードプラン: 月30,000通 ¥15,000/月
└── ★ まず無料プランで開始し、友だち数に応じてアップグレード

拡張ツールの選択肢:

Lステップ:
├── 料金: ¥2,980/月〜
├── 機能: ステップ配信・タグ管理・シナリオ分岐・リッチメニュー
├── メリット: 高機能・柔軟なシナリオ設計
├── デメリット: 学習コストがやや高い
└── ★ 本格的にLINEマーケティングを行う場合に推奨

エルメ（L Message）:
├── 料金: 無料プラン（月1,000通）あり / 有料¥10,780/月〜
├── 機能: ステップ配信・タグ・アンケート・予約管理
├── メリット: 無料から始められる
└── ★ コストを抑えたい場合の選択肢

プロライン:
├── 料金: 無料プラン（月1,000通）あり / 有料¥9,680/月〜
├── 機能: ステップ配信・タグ・フォーム・決済連携
└── ★ シンプルに始めたい場合

推奨:
├── 友だち0-500人: LINE公式アカウントのみ（無料/ライトプラン）
├── 友だち500-2,000人: Lステップ or エルメ（無料プラン）
├── 友だち2,000人+: Lステップ（有料プラン）
└── ★ 最初は最低限のツールで、友だち数に応じてスケール
```

## フロー分析・設計

```
LINE導線のフロー設計:

入口（友だち追加）:
├── Meta広告 → ティザーLP → LINE追加
├── Amazon同梱カード → QRコード → LINE追加
├── SNS（Instagram/X）→ プロフィールリンク → LINE追加
├── 自社EC → ポップアップ → LINE追加
└── ★ 複数の入口を用意し、流入元をタグで管理

自動応答（友だち追加直後）:
├── ウェルカムメッセージ
│   ├── 挨拶 + ブランド紹介
│   ├── 登録特典の案内（クーポン等）
│   └── 次のアクション誘導
├── アンケート
│   ├── 購入経験の有無
│   ├── 興味のある商品カテゴリ
│   └── ★ 回答をタグとして保存 → セグメント配信に活用
└── 初回特典の配布
    ├── クーポンコード
    └── 限定コンテンツ

ステップ配信:
├── Day 0: ウェルカム + 特典
├── Day 1: ブランドストーリー
├── Day 3: 商品の特徴・こだわり
├── Day 5: お客様の声 / レビュー紹介
├── Day 7: 商品購入のご案内
└── ★ 段階的に信頼を構築してから購入を促す

セグメント配信:
├── タグに基づくセグメント
│   ├── 購入者 / 未購入者
│   ├── 流入元（広告 / SNS / 同梱カード）
│   ├── 興味カテゴリ
│   └── 配信反応（開封/クリック）
├── セグメント別の配信内容
│   ├── 購入者 → レビュー依頼 / 新商品案内 / リピート促進
│   ├── 未購入者 → 商品紹介 / 限定オファー
│   └── 高反応者 → VIP特典 / 先行案内
└── ★ 全員に同じ配信ではなく、状況に応じた最適化
```

## タグ管理

```
タグ設計のフレームワーク:

カテゴリ別タグ:

1. 流入元タグ:
├── 「流入:Meta広告」
├── 「流入:同梱カード」
├── 「流入:SNS」
├── 「流入:自社EC」
└── ★ どの経路からの友だちが購入率が高いか分析

2. ステータスタグ:
├── 「ステータス:未購入」
├── 「ステータス:購入済み」
├── 「ステータス:リピーター」
└── ★ 購入ステータスに応じた配信を自動化

3. 興味タグ:
├── 「興味:商品A」
├── 「興味:商品B」
├── 「興味:新商品」
└── ★ アンケート回答や行動（リンククリック）で自動付与

4. 行動タグ:
├── 「行動:リンクA_クリック」
├── 「行動:クーポン使用」
├── 「行動:レビュー投稿」
└── ★ 配信内のリンククリックで自動付与

5. セールタグ:
├── 「セール:PrimeDay告知済み」
├── 「セール:BF告知済み」
└── ★ 重複配信の防止

タグ活用例:
├── 「流入:Meta広告」 AND 「ステータス:未購入」 → 再アプローチ配信
├── 「ステータス:購入済み」 AND 「行動:レビュー未投稿」 → レビュー依頼
└── 「興味:新商品」 → 新商品リリース時の先行告知
```

## リッチメニュー設計

```
リッチメニューの構成:

6分割レイアウト（推奨）:
┌──────────┬──────────┬──────────┐
│ 商品一覧  │ 新商品   │ セール情報 │
├──────────┼──────────┼──────────┤
│ クーポン  │ レビュー │ お問合せ  │
└──────────┴──────────┴──────────┘

各ボタンのアクション:
├── 商品一覧: Amazon ストアフロントへのリンク
├── 新商品: 新商品ページへのリンク（or 準備中メッセージ）
├── セール情報: 次回セール情報のメッセージ表示
├── クーポン: 限定クーポンコードの表示
├── レビュー: レビュー投稿方法の案内
└── お問合せ: 問い合わせフォーム or テキスト入力

デザインのポイント:
├── ブランドカラーを統一
├── アイコンは視認性が高いものを使用
├── テキストは最小限（大きく読みやすく）
├── モバイルでの表示サイズを確認
└── ★ 季節やセール時にリッチメニューを差し替え

状況別リッチメニュー:
├── 通常時: 標準の6分割メニュー
├── セール前: セール告知を最上段に配置
├── プリローンチ時: 新商品情報を強調
└── ★ 拡張ツール（Lステップ等）ならタグに応じて出し分け可能
```

## 配信スケジュール

```
定期配信のスケジュール:

頻度: 週1-2回（配信過多による友だち解除を防止）

配信内容のローテーション:
├── Week 1: 商品の使い方・活用Tips
├── Week 2: お客様の声・レビュー紹介
├── Week 3: 開発裏話・ブランドストーリー
├── Week 4: クーポン・特別オファー
└── ★ 毎回「売り込み」ではなく、価値提供を中心に

配信のルール:
├── 配信時間: 12:00 or 20:00（開封率が高い時間帯）
├── テキスト: 短く簡潔に（スクロールさせない）
├── 画像: リッチメッセージを活用（視覚的に訴求）
├── CTA: 1配信につき1アクション（リンク1つ）
└── ★ 購入を促す配信は月1-2回に留める（過剰な売り込み厳禁）

イベント配信:
├── 大型セール前: 1週間前 + 前日 + 当日
├── 新商品リリース: 1週間前から毎日（ステップ配信）
├── レビュー依頼: 商品到着後5-7日後
└── ★ イベント時は一時的に配信頻度を上げる
```

## 効果測定

```
LINE運用のKPI:

基本指標:
├── 友だち数: 月間追加数 / 累計友だち数
├── ブロック率: 20%以下を維持（配信過多の場合上昇）
├── 開封率: 60%以上（LINE平均は80%前後だが計測方法に注意）
├── クリック率: 5-15%（配信内容による）
└── 購入転換率: LINE経由の購入 / 配信数

プリローンチ用KPI:
├── LINE友だちCPA: ¥300-800
├── 友だち → 購入転換率: 10-20%
├── リリース初日注文数
└── プリローンチROI

効果測定の方法:
├── UTMパラメータ付きリンクでLINE経由の流入を追跡
├── LINE専用クーポンコードで売上を計測
├── 拡張ツールのダッシュボードで配信効果を分析
└── ★ 「LINE経由でどれだけ売上に貢献しているか」を数値化する
```
