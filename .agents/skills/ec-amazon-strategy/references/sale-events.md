# 大型セール施策 詳細ガイド（Amazon物販3ステップメソッド）

## Amazonセールカレンダー

```
年間の主要セールイベント:

├── 1月: 初売りセール（1/2-1/5頃）
│   └── 年始の購買意欲が高い時期
│
├── 3月: 新生活セール（3月下旬）
│   └── 引越し・新学期需要
│
├── 6月: Amazon スマイルSALE（6月上旬）
│   └── 夏前の需要期
│
├── 7月: Prime Day（7月中旬 ※会員限定）
│   └── ★ 年間最大級のセール。最重要イベント
│
├── 9月: Amazon スマイルSALE（9月上旬）
│   └── 秋の需要期
│
├── 10月: Prime感謝祭（10月中旬 ※会員限定）
│   └── 年末商戦の前哨戦
│
├── 11月: ブラックフライデー（11月第4金曜日〜）
│   └── ★ Prime Dayと並ぶ年間最大セール
│
├── 12月: サイバーマンデー / 年末セール
│   └── クリスマス・年末需要
│
└── 不定期: タイムセール祭り（月1-2回）
    └── 数日間の集中セール

★ Prime Day と ブラックフライデーが最重要
★ セール前2-4週間からの準備が成功の鍵
```

## セール準備のタイムライン

```
大型セール準備スケジュール（4週間前から）:

4週間前:
├── □ 在庫の確認・追加発注の判断
│   ├── セール期間中の想定販売数 = 通常の2-5倍
│   ├── FBA入庫のリードタイム（国内7-14日）を逆算
│   └── ★ 在庫切れはセール最大の機会損失
├── □ セール参加の申請（タイムセール等）
│   ├── セラーセントラル → 広告 → ディール
│   └── 申請期限に注意（通常セール2-4週間前が締切）
└── □ 目標設定（売上目標・広告予算・利益目標）

3週間前:
├── □ カタログの最終チェック・改善
│   ├── メイン画像の品質確認
│   ├── タイトル・箇条書きの最適化
│   └── A+コンテンツの更新
├── □ クーポン/プロモーションの設計
│   ├── 値引き率の決定（利益を確保しつつ魅力的な額）
│   └── クーポンバッジの表示確認
└── □ 広告予算の計画
    ├── セール期間中は通常の2-3倍の予算を確保
    └── キャンペーンの事前準備

2週間前:
├── □ 広告キャンペーンの準備
│   ├── セール用キャンペーンの作成（事前に有効化しない）
│   ├── 入札額の事前設定
│   └── ブランド広告（SB/SD）の追加検討
├── □ FBA在庫の入庫確認
│   └── 入庫完了 & 販売可能状態であることを確認
└── □ プリローンチ（LINE/メルマガ）でのセール事前告知

1週間前:
├── □ 最終在庫確認
├── □ クーポン/ディールの設定完了確認
├── □ 広告の最終準備
├── □ LINE/メルマガでセール予告配信
└── □ リアルタイム監視の体制準備

セール当日:
├── □ 朝一: 広告予算・入札の調整
├── □ 午前: セール開始確認・広告有効化
├── □ 随時: 在庫数・注文数の監視
├── □ 夕方: 中間データ確認・入札調整
└── □ 夜: 1日目の結果確認・翌日の調整
```

## セール広告戦略

```
セール期間中の広告運用:

予算:
├── 通常の2-3倍に増額
├── 日予算が早期に消化される場合は追加増額
├── ★ セール中はCPCが上がるため、予算不足に注意

入札戦略:
├── メインKW: 入札を20-50%UP
│   └── セール中は競合も入札を上げるため
├── ブランドKW: 入札を維持 or 微増
│   └── 指名検索を確実に獲得
├── 競合ASIN: 入札を30-50%UP
│   └── セール中の顧客奪取チャンス
└── ★ セール終了後は入札を通常に戻すこと

キャンペーン構成:
├── 通常キャンペーン: 継続（予算増額）
├── セール専用キャンペーン: 期間限定で追加
│   ├── 「セール」「Prime Day」等のKWを追加
│   ├── 競合ASINターゲティングを強化
│   └── SB広告でブランド訴求
└── セール後: セール専用キャンペーンを停止 or 予算縮小

注意点:
├── ❌ セール中にCPCだけで判断して入札を下げる
├── ❌ 予算不足で午前中に配信停止
├── ✅ 一時的なCPC上昇は想定内
├── ✅ セール中の売上 → SEOスコア大幅UP → セール後の恩恵
└── ★ セール中の広告は「投資」。セール後のSEO効果で回収
```

## セール値引き戦略

```
値引き手法の選択:

1. タイムセール（Lightning Deal）
├── 申請: セラーセントラルから事前申請
├── 費用: ¥4,000-10,000/回（セールにより異なる）
├── 効果: タイムセールページに掲載 → 大幅な露出増
├── 値引き: Amazon指定の最低割引率（通常15-25%以上）
└── ★ 申請が通れば効果大。枠に限りあり

2. ベストディール（7日間ディール）
├── Prime Day等の大型セールで利用可能
├── 期間: セール期間中の7日間
├── バッジ: 商品に「ディール」バッジが表示
├── 値引き: 通常15-20%以上
└── ★ タイムセールより長期間露出される

3. クーポン
├── 自分で設定可能（申請不要）
├── バッジ: 検索結果に「クーポン」バッジ表示
├── 費用: クーポン値引き分 + ¥60/回の手数料
├── 柔軟性: 割引率・期間を自由に設定
└── ★ タイムセールに参加できない場合の代替策

4. ポイント付与
├── 自分で設定可能
├── ポイント数に応じて検索結果に表示
├── 実質値引きだが、表示上の売価は維持
└── ★ 値引きせずにお得感を出す方法

値引き率の決定:
├── 利益率30%以上を維持できる範囲
├── 競合のセール値引きを参考
├── 一般的: 10-20%OFF（タイムセール以外）
├── タイムセール: 15-30%OFF（Amazon指定に従う）
└── ★ 過度な値引きは利益を圧迫。ブランド価値も低下
```

## セール後の対応

```
セール終了後のオペレーション:

即座にやること:
├── 広告入札を通常に戻す
├── セール用キャンペーンを停止
├── クーポン/プロモーションを終了
├── 在庫残数の確認
└── セール結果のデータ収集

1週間以内:
├── セール結果の分析レポート作成
│   ├── 売上 / 注文数 / 平均注文単価
│   ├── 広告費 / ACoS / TACoS
│   ├── セール前後の検索順位変動
│   └── セール前後のオーガニック売上比較
├── 在庫補充の判断
│   └── セールで在庫が減った場合は即座に発注
└── レビュー獲得施策の開始
    └── セールの大量購入者にレビューリクエスト

セール後のSEO効果:
├── セール中の大量注文 → 販売速度スコアが大幅UP
├── 検索順位の上昇（セール後1-2週間で効果が出る）
├── オーガニック売上の増加
├── ★ セール後2-4週間が「SEOボーナス期間」
└── ★ この期間に広告を維持して順位を固定する

次回セールへの改善:
├── 何が良かったか / 改善点は何か
├── 在庫は足りたか / 余ったか
├── 広告戦略は適切だったか
├── 値引き率は最適だったか
└── ★ 毎回の学びを次のセールに活かす
```

## セール別の特性

```
主要セール別の攻略ポイント:

Prime Day:
├── 特性: Amazon最大級。Prime会員限定
├── 購買意欲: 非常に高い（セール目当ての来訪）
├── CPC: 通常の2-3倍に上昇
├── 戦略: 在庫多め確保 + 積極的な値引き + 広告予算大幅増
└── ★ 年間売上の10-15%がこの1-2日に集中することも

ブラックフライデー:
├── 特性: Prime Dayと並ぶ大型セール
├── 購買意欲: 非常に高い（年末商戦の開始）
├── CPC: 通常の2-3倍
├── 戦略: ギフト需要を意識した訴求 + クリスマス向け画像
└── ★ クリスマスプレゼント需要を取り込む

タイムセール祭り:
├── 特性: 月1-2回の定期セール
├── 購買意欲: 中〜高
├── CPC: 通常の1.3-1.5倍
├── 戦略: 在庫調整に活用 + 新商品の認知拡大
└── ★ 大型セールほどの効果はないが、継続参加で認知向上

新生活セール:
├── 特性: 3月下旬の季節セール
├── 購買意欲: カテゴリによる（生活用品が強い）
├── CPC: 通常の1.2-1.5倍
├── 戦略: 新生活に関連する訴求（新居・新学期等）
└── ★ 生活用品・家電・収納カテゴリに特に効果的
```
