# プリローンチ（事前集客）詳細ガイド（Amazon自社ブランド戦略フレームワーク）

## プリローンチの概要

```
プリローンチとは:
├── 商品発売前にMeta広告→LP→LINE/メルマガで見込み客を事前に集める戦略
├── リリース日に一斉告知 → 初日から大量注文 → SEOスコアを一気に構築
├── Amazonの実績SEO（注文数・販売速度）を最短で上げる手法
└── 「通常の広告運用だけでは3-4ヶ月かかるSEO構築を、初日で加速する」

プリローンチの流れ:
Meta広告 → ティザーLP → LINE友だち追加 → ステップ配信 → リリース日に一斉購入告知

メリット:
├── 初日から注文が集中 → Amazonが「売れる商品」と判断
├── SEOスコアが一気に上がる → オーガニック検索順位の早期確保
├── レビューの早期獲得（購入者 → レビュー依頼）
├── 広告費の効率化（SEO上位なら広告依存度が下がる）
└── 競合より早くポジションを確立

デメリット・リスク:
├── 事前の広告費が先行投資になる（回収に時間がかかる場合あり）
├── LINE友だち数が想定を下回ると効果半減
├── 商品品質が伴わないとレビューで逆効果
└── Meta広告のCPA次第で投資対効果が変動
```

## プリローンチの判断基準

```
プリローンチを実施すべき場合:
├── ✅ 競合が強い市場（レビュー数が多い競合がいる）
├── ✅ 高単価商品（初動の注文価値が大きい）
├── ✅ 差別化ポイントが明確で訴求しやすい
├── ✅ 広告クリエイティブの素材が豊富
└── ✅ LINE/メルマガ運用のリソースがある

プリローンチなしで直リリースすべき場合:
├── ✅ 競合が弱い市場（レビュー少ない）
├── ✅ 低単価商品（CPA回収が難しい）
├── ✅ 広告予算が限られている
├── ✅ 初めてのAmazon出品（まずシンプルに始める）
└── ✅ 2商品目以降で既存LINEリストがある場合

★ 初商品は直リリースで学び、2商品目以降でプリローンチを活用するのが推奨
```

## Step 1: Meta広告の設計

```
プリローンチ用Meta広告:

目的: ティザーLPへの誘導 → LINE友だち追加
├── キャンペーン目的: リード獲得 or トラフィック
├── ターゲティング: 商品カテゴリに関心のある層
│   ├── 興味関心ターゲティング（関連KW・競合ブランド）
│   ├── 類似オーディエンス（既存顧客がいる場合）
│   └── ブロード（AI最適化に任せる）
│
├── クリエイティブ:
│   ├── 動画: 商品の特徴・使用シーンを15-30秒で訴求
│   ├── 画像: ビフォーアフター・特徴訴求・限定感
│   ├── カルーセル: 複数の特徴を順番に見せる
│   └── ★ 3-5パターンを同時テスト
│
├── コピー:
│   ├── 問題提起型: 「○○で困っていませんか？」
│   ├── 新商品告知型: 「ついに○○が解決する商品が登場」
│   ├── 限定感型: 「先行予約で○○%OFF」
│   └── ★ LP誘導が目的。Amazon購入の話はしない
│
├── CPA目標:
│   ├── LINE友だち追加CPA: ¥300-800（商品単価による）
│   ├── 高単価商品（¥5,000+）: CPA ¥500-800許容
│   ├── 中単価商品（¥2,000-5,000）: CPA ¥300-500
│   └── ★ CPAが合わない場合はクリエイティブを変更
│
└── 予算:
    ├── 目標LINE友だち数から逆算
    ├── 例: 500人 × CPA ¥500 = ¥250,000
    ├── 配信期間: リリース2-4週間前から
    └── ★ 初期は日予算¥3,000-5,000でテスト
```

## Step 2: ティザーLP（ランディングページ）

```
ティザーLPの構成:

目的: Meta広告からの流入者をLINE友だちに変換

構成テンプレート:
├── ① ヘッダー（ファーストビュー）
│   ├── キャッチコピー: 問題提起 or ベネフィット訴求
│   ├── 商品イメージ画像
│   └── CTA: 「先行情報をLINEで受け取る」
│
├── ② 問題提起セクション
│   ├── ターゲットが抱える悩み・課題
│   ├── 共感を得る表現
│   └── 「実は○○が原因でした」
│
├── ③ 解決策セクション
│   ├── 商品の概要紹介
│   ├── 主要な特徴3つ
│   └── 「だから○○を開発しました」
│
├── ④ 限定特典セクション
│   ├── LINE登録者限定クーポン（例: 15%OFF）
│   ├── 先行予約の案内
│   └── 限定数量の告知
│
├── ⑤ CTA（メインのLINE追加ボタン）
│   ├── 「今すぐLINEで先行情報を受け取る」
│   ├── ボタンは大きく目立つデザイン
│   └── LINE友だち追加のQRコード
│
└── ⑥ FAQ（必要に応じて）
    ├── いつ発売ですか？
    ├── LINE登録は無料ですか？
    └── キャンセルは可能ですか？

構築ツール:
├── WordPress + Elementor（推奨）
├── Shopify Landing Page
├── ペライチ
└── ★ モバイルファーストで設計すること
```

## Step 3: LINE構築・ステップ配信

```
プリローンチ用LINE配信シナリオ:

友だち追加時（即座に配信）:
├── ウェルカムメッセージ
├── 自己紹介・ブランドストーリー
├── 「リリース日は○月○日です。楽しみにお待ちください！」
└── アンケート（ニーズの確認）

リリース7日前:
├── 商品の開発ストーリー
├── こだわりポイントの紹介
└── 期待感を醸成する情報

リリース3日前:
├── 商品の詳細スペック
├── 使用レビュー・モニター感想（あれば）
└── 「あと3日で発売開始！」

リリース前日:
├── 最終リマインド
├── 限定クーポンコードの事前配布
├── 「明日○時にAmazonで販売開始します」
└── 購入方法の説明（Amazon商品ページへのリンク）

リリース当日:
├── 「本日発売開始しました！」
├── Amazon商品ページへの直リンク
├── クーポン適用方法の説明
├── 「在庫限定・先着○名様」の限定感
└── ★ 朝・昼・夕の3回配信も検討

リリース翌日:
├── 購入御礼メッセージ
├── レビューのお願い
├── まだ購入していない方へのリマインド
└── 追加特典の案内（あれば）

配信のルール:
├── 画像リッチメッセージを活用（テキストだけより反応率UP）
├── 1通あたりの文字数は短めに（スマホで読みやすく）
├── リッチメニューで商品情報にアクセスしやすくする
└── タグ付けで反応者/未反応者を分けて再アプローチ
```

## Step 4: リリース日オペレーション

```
リリース当日のタイムライン:

事前準備（前日まで）:
├── □ Amazon商品ページが公開状態であることを確認
├── □ FBA在庫が「販売可能」になっていることを確認
├── □ クーポン/プロモーションの設定完了
├── □ SP広告キャンペーンの準備（即時開始設定）
├── □ LINE配信文面の最終確認
└── □ SNS投稿の事前準備

リリース当日:
├── 06:00 Amazon商品ページの最終確認
├── 07:00 SP広告キャンペーンを有効化
├── 08:00 LINE一斉配信（第1弾）
├── 09:00 SNS投稿（Instagram/X）
├── 12:00 LINE追加配信（まだ購入していない方向け）
├── 15:00 セッション数・注文数・広告データの中間確認
├── 18:00 LINE追加配信（ラストチャンス訴求）
├── 21:00 1日目の結果まとめ・翌日の計画調整
└── ★ リアルタイムでセラーセントラルを監視

初日の目標:
├── LINE配信→購入の転換率: 10-20%
├── 例: LINE友だち500人 → 50-100注文
├── 検索順位の変動をモニタリング
└── レビュー依頼の開始準備

初日以降（Week 1）:
├── 毎日の注文数・検索順位を追跡
├── 広告の初期データで入札調整
├── レビュー獲得施策の開始
├── 追加のLINE配信（まだ購入していない方向け）
└── SNSでの口コミ促進
```

## リスク管理

```
プリローンチのリスクと対策:

リスク1: LINE友だちが集まらない
├── 原因: Meta広告のCPAが高い / クリエイティブの訴求力不足
├── 対策: クリエイティブの差し替え / ターゲティング変更
├── 判断基準: 1週間でCPA目標の2倍以上 → 戦略見直し
└── 最悪のケース: プリローンチ中止 → 直リリースに切替

リスク2: LINE登録はするが購入しない
├── 原因: 配信内容が魅力不足 / 価格が高い / 購入動線が不明確
├── 対策: 配信内容の改善 / クーポン強化 / 購入方法の丁寧な説明
└── 目標転換率: 最低10%、理想20%

リスク3: 広告費が回収できない
├── 原因: CPA高 × 転換率低 = ROIマイナス
├── 対策: プリローンチのROIを事前にシミュレーション
├── 許容範囲: 広告費は3ヶ月以内にSEO効果で回収
└── ★ プリローンチの広告費は「SEO投資」と位置づける

リスク4: 在庫切れ
├── 原因: 想定以上の注文
├── 対策: 初回発注量をプリローンチ目標の1.5倍に設定
└── FBA在庫が切れるとSEOスコアが急落するので要注意

投資回収シミュレーション:
├── 例: プリローンチ広告費 ¥200,000
├── LINE友だち: 400人（CPA ¥500）
├── リリース日注文: 60件（転換率15%）
├── 商品単価¥3,000 × 60件 = ¥180,000（初日売上）
├── SEO効果でオーガニック売上が月¥300,000増加
├── 3ヶ月でSEO効果による追加売上: ¥900,000
└── ROI: (¥900,000 - ¥200,000) / ¥200,000 = 350%
```
