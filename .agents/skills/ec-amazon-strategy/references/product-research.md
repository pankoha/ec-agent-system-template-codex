# 商品リサーチ 詳細ガイド（Amazon物販3ステップメソッド）

## 基本思想: レッドオーシャン戦略

```
なぜレッドオーシャンを狙うのか:
├── ブルーオーシャン（競合がいない市場）は「需要がない」可能性が高い
├── レッドオーシャン（競合が多い市場）= 需要が証明されている市場
├── 大きな市場で差別化して勝つ方が、小さな市場を独占するより利益が大きい
└── 重要: 「競合が多い」≠「勝てない」。差別化できれば十分に戦える
```

## Step 1: 自己分析（リサーチ前の準備）

```
商品選定の前に自分を知る:
├── 興味・関心のある分野
│   ├── 日常的に使う商品カテゴリ
│   ├── 趣味・スポーツ関連
│   └── 仕事・専門知識のある分野
│
├── 強み・リソース
│   ├── 業界知識・専門性
│   ├── サプライヤーとのコネクション
│   ├── 資金力（初期投資可能額）
│   └── 時間的リソース
│
└── 制約条件
    ├── 初期投資の上限
    ├── 在庫保管スペース
    ├── 取り扱い不可カテゴリ（危険物・食品等の規制）
    └── 競合知識の有無
```

## Step 2: 初期選定基準（5つのフィルター）

```
すべてのフィルターを通過した商品のみ候補とする:

フィルター1: 月販数量 ≥ 100個
├── Keepa / Helium 10 / Jungle Scout等で確認
├── ベンチマーク商品（上位10商品）の平均月販
├── 100個未満の市場は規模が小さすぎる
└── 理想: 月販300〜1,000個の市場

フィルター2: 単品利益 ≥ ¥500
├── 売価 - 原価 - FBA手数料 - 販売手数料 - 広告費
├── ¥500未満だと広告費を捻出できない
└── 理想: 単利¥1,000〜¥3,000

フィルター3: 利益率 ≥ 30%
├── 利益 / 売価 × 100
├── 30%未満だとAmazon手数料と広告費で赤字リスク
├── Amazon販売手数料（8-15%）+ FBA手数料 を考慮
└── 理想: 40%以上

フィルター4: 通年商品である
├── 季節商品は在庫リスクが高い
├── Keepaのグラフで年間の需要変動を確認
├── 夏物/冬物は避けるか、年間で複数商品でカバー
└── 例外: 季節商品でも需要期が6ヶ月以上ならOK

フィルター5: ベンチマークを超えられる
├── 既存商品のレビュー（特に低評価）を分析
├── 改善可能な弱点が明確に存在する
├── OEM/ODMで実現可能な改善である
└── 価格据え置き（or わずかなUP）で差別化できる
```

## Step 3: 市場調査の実践手順

### 3-1: カテゴリリサーチ
```
Amazonでの調査手順:
├── Step A: ベストセラーランキングを確認
│   ├── Amazon.co.jp → ランキング → カテゴリ別
│   ├── 興味のあるカテゴリのTOP 100を確認
│   └── 自社ブランド（中国OEM以外も含む）が多いカテゴリに注目
│
├── Step B: Keepaでデータ確認
│   ├── 売れ筋商品の価格推移
│   ├── ランキング推移（売上の安定性）
│   ├── 出品者数の推移
│   └── 在庫切れの頻度（需要 > 供給のサイン）
│
├── Step C: 検索ボリュームの確認
│   ├── Helium 10 / Jungle Scout のキーワードツール
│   ├── メインキーワードの月間検索数
│   ├── 関連キーワードの洗い出し
│   └── 検索ボリュームのトレンド（増加傾向が理想）
│
└── Step D: 競合の強さを評価
    ├── TOP 10商品のレビュー数（1,000件超は強敵）
    ├── レビュー平均評価（4.5以上は品質が高い）
    ├── ブランド登録の有無
    └── A+コンテンツの充実度
```

### 3-2: 競合レビュー分析（最重要）
```
レビュー分析の手順:
├── ベンチマーク商品のレビュー（特に★1〜3）を徹底的に読む
│
├── 不満点をカテゴリ分類:
│   ├── 品質問題: 壊れやすい・素材が安い・仕上げが粗い
│   ├── 機能不足: ○○ができない・○○がついていない
│   ├── サイズ問題: 大きすぎる/小さすぎる・想像と違った
│   ├── 使い勝手: 操作が難しい・説明書がわかりにくい
│   ├── デザイン: 見た目が安い・色味が写真と違う
│   └── パッケージ: 梱包が雑・ギフトに使えない
│
├── 改善優先度の判定:
│   ├── 高: 複数レビューで共通する不満 + OEMで改善可能
│   ├── 中: 一部レビューでの不満 + 改善コストが適正
│   └── 低: 個人的な好み + 改善コストが高い
│
└── 差別化ポイントの決定:
    ├── TOP 3の改善ポイントを特定
    ├── それぞれのOEM改善コストを概算
    ├── 価格への影響を計算
    └── 「この改善なら○○円UPで実現可能」を明確化
```

## Step 4: 原価計算テンプレート

```
Amazon自社ブランド 原価計算シート:

【売価設定】
├── 想定売価: ¥______
├── 競合価格帯: ¥______ 〜 ¥______
└── ポジショニング: 低価格 / 中価格 / 高価格

【コスト構造】
├── 商品原価（FOB）: ¥______
├── 国際送料: ¥______
├── 関税（HSコード: ______）: ¥______
├── 国内送料（FBA納品）: ¥______
├── パッケージ・ラベル: ¥______
├── 検品費用: ¥______
├── Amazon販売手数料（___%）: ¥______
├── FBA手数料: ¥______
└── 総コスト: ¥______

【利益計算】
├── 粗利: ¥______ （売価 - 総コスト）
├── 粗利率: ______%
├── 広告費想定（売上の___%）: ¥______
├── 純利益: ¥______
└── 純利益率: ______%

【判定基準】
├── □ 単利 ≥ ¥500
├── □ 利益率 ≥ 30%（広告費前）
├── □ 広告費15%込みでも利益が出る
└── □ 価格改定の余地がある（原価率50%以下が理想）
```

## Step 5: 商品スコアリングマトリクス

```
候補商品を100点満点でスコアリング:

| 評価項目 | 配点 | 基準 |
|---------|------|------|
| 市場規模 | 20点 | 月販100-300=10, 300-500=15, 500+=20 |
| 利益率 | 20点 | 30-35%=10, 35-40%=15, 40%+=20 |
| 差別化可能性 | 20点 | 明確な改善点の数と実現可能性 |
| 競合の強さ | 15点 | レビュー100未満=15, 100-500=10, 500+=5 |
| 通年需要 | 10点 | 月別変動率が20%以下=10, 30%以下=7, 30%超=3 |
| 参入コスト | 10点 | MOQ×単価が30万以下=10, 30-50万=7, 50万超=3 |
| 自身の知識 | 5点 | カテゴリに詳しい=5, 普通=3, 全く知らない=1 |

判定:
├── 80点以上: 最優先候補。即座にサプライヤー調査開始
├── 60-79点: 有望候補。追加調査で判断
├── 40-59点: 条件付き候補。特定項目の改善余地を検討
└── 40点未満: 見送り。他の候補を優先
```

## よくある失敗パターン

```
商品リサーチでの典型的な失敗:

❌ 失敗1: 「自分が好きだから」だけで選ぶ
   └── ✅ 対策: 必ず5つのフィルターを通す。感情ではなくデータで判断

❌ 失敗2: 競合が少ない市場を狙う（ブルーオーシャン幻想）
   └── ✅ 対策: 競合が少ない = 需要が少ない可能性。月販100以上を確認

❌ 失敗3: 原価だけ見て利益率を計算する
   └── ✅ 対策: FBA手数料・販売手数料・広告費・関税すべて含めて計算

❌ 失敗4: ベンチマークなしに参入する
   └── ✅ 対策: 「この商品のここを改善する」が言えない商品は見送り

❌ 失敗5: 最初から複数商品に手を出す
   └── ✅ 対策: まず1商品で成功パターンを確立してから2商品目

❌ 失敗6: 季節商品に初参入する
   └── ✅ 対策: 初商品は通年需要の商品を選ぶ
```
