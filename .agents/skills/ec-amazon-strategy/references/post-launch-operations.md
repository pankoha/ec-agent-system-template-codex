# 販売後の運用 詳細ガイド（Amazon物販3ステップメソッド）

## 販売後の運用概要

```
発売後は「放置」ではなく「継続的な最適化」が必要:
├── 日次: 数値確認・異常検知
├── 週次: 広告チューニング・在庫確認
├── 月次: カタログ改善・戦略見直し
├── 四半期: 大型セール準備・新商品検討
└── ★ 「売れる仕組み」を作り、維持・強化し続ける
```

## 日次モニタリング

```
毎日チェックする指標:

セラーセントラル → ビジネスレポート:
├── セッション数: 商品ページの訪問数
│   └── 急減 → 検索順位下落? 広告停止? 在庫切れ?
├── ページビュー数: ページの閲覧回数（バリエーション含む）
├── 注文数: 当日の注文数
│   └── 通常の±30%以上の変動 → 要因分析
├── CVR（ユニットセッション率）: 訪問者のうち購入した割合
│   └── 急低下 → 価格変動? 競合新商品? レビュー悪化?
├── 売上: 当日の売上金額
├── 広告データ:
│   ├── インプレッション / クリック / CPC
│   ├── 広告売上 / ACoS
│   └── 予算消化状況
└── 在庫数: FBA在庫の残数
    └── 安全在庫を下回りそう → 補充発注

日次チェックのルーティン:
├── 09:00 前日のデータ確認（セラーセントラル）
├── 09:15 異常値の有無をチェック
├── 09:30 広告予算の消化状況確認
└── ★ 1日15-30分で完了する仕組みを作る
```

## 週次オペレーション

```
毎週実施すること:

月曜日: 前週の週次レポート作成
├── 売上/セッション/CVR/注文数の週次推移
├── 広告KPIの週次推移（ACoS/TACoS/ROAS）
├── キーワードランキングの変動
└── 競合の動向チェック

水曜日: 広告チューニング（8ステップ）
├── 全体数値 → キャンペーン → ターゲティング → 思考
├── オーガニック込み → 広告順位 → 自然検索順位 → 拡張
└── ★ 毎週同じ曜日に実施する（習慣化）

金曜日: 在庫・物流チェック
├── FBA在庫残数の確認
├── 販売速度から在庫切れ予測日を計算
├── 補充が必要な場合は発注手配
├── 長期保管手数料の対象在庫チェック
└── ★ 在庫切れはSEOの致命傷。余裕を持って発注
```

## 月次戦略レビュー

```
月次で見直すこと:

1. P&L（損益計算）の確認
├── 月間売上
├── 売上原価（商品原価 + FBA手数料 + 販売手数料）
├── 広告費
├── その他経費
├── 営業利益 / 利益率
└── ★ 利益率30%以上を維持できているか

2. SEO状態の確認
├── 主要KW（5-10個）のランキング推移
├── オーガニック売上比率の推移
├── TACoSの推移
├── 新規獲得KWの有無
└── ★ オーガニック比率70%以上が目標

3. カタログ改善の検討
├── CTR/CVRの推移分析
├── ABテストの結果確認
├── レビュー分析に基づく改善点
├── 新しい画像/テキストのテスト計画
└── A+コンテンツの更新

4. 競合モニタリング
├── 新規参入者の有無
├── 競合の価格変動
├── 競合の新商品・カタログ変更
├── 競合のレビュー動向
└── ★ 競合が強化している分野を把握し、先手を打つ

5. 次月の計画
├── 広告予算の調整
├── 在庫補充計画
├── カタログ改善計画
├── 新商品の検討状況
└── 大型セールの準備（該当月の場合）
```

## 在庫管理

```
FBA在庫管理の基本:

安全在庫の計算:
├── 安全在庫 = 1日平均販売数 × リードタイム（日数）× 1.5
├── リードタイム: 発注→FBA入庫完了までの日数
│   ├── 国内仕入れ: 7-14日
│   ├── 海外OEM: 30-60日（製造+輸送+通関+国内配送）
│   └── 繁忙期: 通常+7-14日余裕を見る
└── 例: 日販10個 × リードタイム45日 × 1.5 = 675個の安全在庫

在庫切れの影響:
├── SEOスコアの急落（販売速度がゼロになる）
├── 検索順位の下落（復活に2-4週間かかる場合あり）
├── レビュー獲得の機会損失
├── 広告の強制停止
├── 競合への顧客流出
└── ★ 在庫切れは「最大の損失」。予防が最優先

在庫過剰の影響:
├── FBA長期保管手数料（365日超の在庫に課金）
├── キャッシュフローの悪化
├── 廃棄・返送コスト
└── 対策: 販売予測に基づく適正在庫管理

FBA補充サイクル:
├── 週次: 在庫残数と販売速度をチェック
├── 発注判断: 安全在庫を下回りそうなタイミングで発注
├── 大型セール前: 通常の2-3倍の在庫を確保
└── 新商品: 初回ロットは1-2ヶ月分から開始
```

## 価格戦略

```
Amazon価格戦略:

基本原則:
├── 価格は競合と同等 or やや高め（差別化で勝つ）
├── 安売り競争に入らない
├── クーポンやポイントで実質価格を調整
└── 利益率30%以上を維持

価格設定の考え方:
├── 原価率: 30-40%が理想
├── Amazon手数料: 15-25%（販売手数料+FBA）
├── 広告費: 10-15%（安定期）
├── 利益: 30-40%
└── 例: 売価¥3,000 → 原価¥900 + 手数料¥600 + 広告¥450 = 利益¥1,050（35%）

クーポン活用:
├── パーセントオフクーポン: 5-15%が一般的
├── 金額オフクーポン: ¥100-500 OFF
├── 効果: 検索結果にクーポンバッジ表示 → CTR向上
├── コスト: クーポン値引き分 + ¥60/回の手数料
└── ★ 常時クーポンより、期間限定で効果的に使う

値上げの判断:
├── レビュー20件以上 & 平均4.0以上
├── オーガニック検索順位が安定（TOP 10以内）
├── CVRが安定（5%以上）
├── 在庫が安定して供給できる
└── ★ 段階的に100-200円ずつ上げ、CVRへの影響を観察
```

## 2商品目以降の投入

```
新商品投入の判断基準:

投入タイミング:
├── 1商品目が安定期に入った（オーガニック比率50%以上）
├── 広告チューニングがルーティン化できている
├── 在庫管理が安定している
├── 資金に余裕がある（1商品目の利益で投資可能）
└── ★ 1商品目が不安定な状態で手を広げない

2商品目の選択肢:
├── パターンA: 同カテゴリの関連商品
│   ├── メリット: 既存知識・サプライヤーが活用できる
│   ├── メリット: クロスセル・SB広告で相乗効果
│   └── 例: マグカップ → タンブラー → 水筒
│
├── パターンB: 同カテゴリのバリエーション追加
│   ├── メリット: 既存カタログにバリエーション追加
│   ├── メリット: レビューの統合が可能
│   └── 例: 500ml → 350ml / 新色追加
│
├── パターンC: 全く別のカテゴリ
│   ├── メリット: リスク分散
│   ├── デメリット: ゼロからの学習コスト
│   └── ★ 2商品目はパターンA or Bが推奨
│
└── 投入ペース:
    ├── 年間3-5商品が現実的なペース（個人〜少人数）
    ├── 各商品に十分なリソースを投下
    └── ★ 「薄く広く」より「深く狭く」で各商品を育てる
```

## 競合モニタリング

```
継続的な競合分析:

監視項目:
├── 価格変動: 週次で競合TOP5の価格をチェック
├── 新商品: 新規参入・新商品の登場
├── カタログ変更: 画像・タイトル・A+の変更
├── レビュー動向: レビュー数・評価の推移
├── 広告戦略: スポンサー広告の出稿状況
└── 在庫状況: 在庫切れ（チャンスの可能性）

競合への対応:
├── 価格を下げてきた場合:
│   ├── ❌ 即座に追随値下げ（価格競争に巻き込まれる）
│   ├── ✅ 自社の差別化ポイントを強化
│   ├── ✅ クーポンやポイントで対応（一時的な値下げ）
│   └── ✅ A+コンテンツ・画像の改善で価値を訴求
│
├── 新規参入者が現れた場合:
│   ├── 冷静にスペック・価格を分析
│   ├── 自社のレビュー蓄積（先行者優位）を活かす
│   ├── カタログのさらなる改善
│   └── ★ 先にポジションを確立していれば大きな脅威にはならない
│
└── 競合が在庫切れの場合:
    ├── 一時的にシェア拡大のチャンス
    ├── 広告を強化（競合のKWを狙う）
    └── ★ ただし自社の在庫管理は冷静に判断
```

## アカウントヘルスの維持

```
Amazonアカウントの健全性管理:

重要指標:
├── 注文不良率: 1%未満を維持
│   ├── A-to-Z保証クレーム
│   ├── クレジットカードのチャージバック
│   └── ネガティブフィードバック
├── 出荷前キャンセル率: 2.5%未満
├── 出荷遅延率: 4%未満（FBAなら基本問題なし）
└── 在庫パフォーマンス指標（IPI）: 400以上を維持

アカウント停止リスクの回避:
├── ❌ 偽レビューの購入
├── ❌ 規約違反の商品同梱カード
├── ❌ 商品の品質に関する深刻な問題の放置
├── ❌ 知的財産権の侵害
├── ✅ 規約を定期的に確認し遵守
├── ✅ 顧客対応は迅速かつ丁寧に
└── ✅ 品質問題は即座に対応（リコール含む）

★ アカウント停止 = 事業の終了。リスク管理は最優先事項
```
