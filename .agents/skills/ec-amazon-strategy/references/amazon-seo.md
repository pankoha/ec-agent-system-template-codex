# Amazon SEO 詳細ガイド（Amazon自社ブランド戦略フレームワーク）

## Amazon SEOの2軸構造

```
Amazon検索アルゴリズム（A10）は2つの軸で順位を決定する:

┌─────────────────────────────────────────────────┐
│              Amazon SEO = テキストSEO × 実績SEO    │
├─────────────────────────────────────────────────┤
│                                                   │
│  テキストSEO（インデックス）                       │
│  ├── 「何のキーワードで表示されるか」を決める       │
│  ├── タイトル・箇条書き・説明文・バックエンドKW     │
│  ├── A+コンテンツ（ブランドストーリー含む）         │
│  └── 設定すれば即反映。コントロール可能              │
│                                                   │
│  実績SEO（ランキング）                             │
│  ├── 「そのキーワードで何位に表示されるか」を決める │
│  ├── 注文数・レビュー数/評価・CTR・CVR・セッション  │
│  ├── 蓄積に時間がかかる。広告で加速可能              │
│  └── 「売れている商品がさらに売れる」正のスパイラル  │
│                                                   │
│  ★ テキストSEOで「表示対象」になり                  │
│    実績SEOで「上位表示」される                      │
│  ★ どちらか一方だけでは不十分                      │
└─────────────────────────────────────────────────┘
```

## テキストSEO: 詳細設計

### タイトル最適化
```
タイトルの構成ルール:
├── 文字数: 全角80文字以内（推奨60-70文字）
├── 構成: [ブランド名] [メインKW] [サブKW] [特徴/スペック]
├── 例: 「[ブランド名] 保温マグカップ 真空断熱 500ml ステンレス製 蓋付き オフィス用」
│
├── 優先順位:
│   ├── 1位: メインキーワード（最も検索ボリュームが大きいKW）
│   ├── 2位: サブキーワード（検索意図が異なる2-3語）
│   ├── 3位: 特徴・スペック（サイズ・素材・色等）
│   └── 4位: ブランド名（先頭 or 末尾）
│
├── やるべきこと:
│   ├── ✅ 検索されるキーワードを自然に含める
│   ├── ✅ 競合のタイトルを分析して差別化
│   ├── ✅ 最も重要なKWを先頭に配置
│   └── ✅ スマホでの表示（先頭40文字程度）を意識
│
└── やってはいけないこと:
    ├── ❌ キーワードの羅列（スパム判定リスク）
    ├── ❌ 「最高」「No.1」等の主観的表現
    ├── ❌ 価格情報の記載
    └── ❌ 他ブランド名の使用
```

### 箇条書き（Bullet Points）最適化
```
5つの箇条書きの設計:
├── 各ポイント: 200-250バイト以内
│
├── 構成テンプレート:
│   ├── 1つ目: 最大のベネフィット（メインKW含む）
│   ├── 2つ目: 2番目のベネフィット（サブKW含む）
│   ├── 3つ目: 素材・品質の訴求
│   ├── 4つ目: 使い方・用途の提案
│   └── 5つ目: 保証・サポート情報
│
├── 書き方のルール:
│   ├── ✅ ベネフィット中心（○○だから□□できる）
│   ├── ✅ 各ポイントの冒頭に重要KWを配置
│   ├── ✅ 数字を使って具体的に（「保温6時間」等）
│   └── ✅ 絵文字やマークで視認性UP（【】や★等）
│
└── 避けるべきこと:
    ├── ❌ スペックの羅列のみ
    ├── ❌ 他社比較（「○○社より優れている」等）
    └── ❌ HTMLタグの使用
```

### バックエンドキーワード
```
バックエンド検索キーワードの設定:
├── 文字数上限: 500バイト（日本語約166文字）
├── 用途: 表には見えないが検索対象になるキーワード
│
├── 含めるべきキーワード:
│   ├── タイトル・箇条書きに入りきらなかった関連KW
│   ├── 類義語・別の言い回し
│   ├── ひらがな/カタカナの表記揺れ
│   ├── 用途・シーン（「プレゼント」「オフィス」等）
│   └── 素材・特徴の別表現
│
├── 含めてはいけないもの:
│   ├── ❌ 競合ブランド名（規約違反）
│   ├── ❌ ASIN（規約違反）
│   ├── ❌ 「最安」「激安」等の価格訴求
│   └── ❌ すでにタイトルに含まれているKW（重複は不要）
│
└── テクニック:
    ├── スペースで区切る（カンマは不要）
    ├── 単数形のみでOK（Amazonが自動で複数形も対象）
    └── 定期的に検索トレンドを確認して更新
```

### A+コンテンツ
```
A+コンテンツ（Enhanced Brand Content）:
├── 前提: ブランド登録（Brand Registry）が必要
│
├── 構成要素:
│   ├── ブランドストーリー: ブランドの背景・理念
│   ├── 商品特徴モジュール: 画像+テキストで特徴を訴求
│   ├── 比較表モジュール: 自社商品ラインの比較
│   ├── 技術仕様モジュール: スペック詳細
│   └── FAQ/使い方モジュール: よくある質問と回答
│
├── SEO効果:
│   ├── A+コンテンツ内のテキストはインデックス対象（2024年以降強化）
│   ├── 滞在時間の増加 → CVR向上 → 実績SEOに貢献
│   └── ブランドストーリーで信頼性UP → CVR向上
│
├── 推奨モジュール構成（7枚以内）:
│   ├── モジュール1: ブランドロゴ + ストーリー
│   ├── モジュール2: 最大のベネフィット（画像+テキスト）
│   ├── モジュール3: 3つの特徴（3列画像+テキスト）
│   ├── モジュール4: 使用シーン/ライフスタイル
│   ├── モジュール5: 商品スペック/仕様
│   ├── モジュール6: 自社商品比較表
│   └── モジュール7: FAQ or サポート情報
│
└── 注意点:
    ├── 画像のalt テキストにもKWを含める
    ├── モバイル表示を必ず確認
    └── 定期的に更新（季節やプロモーションに合わせて）
```

## 実績SEO: 向上戦略

### 実績SEOの構成要素
```
実績SEOに影響する要素（重要度順）:

1. 注文数（販売速度）
├── 直近7日間・30日間の注文数が最重要
├── 急増（スパイク）より安定した注文が評価される
├── 広告経由でも注文数としてカウントされる
└── ★ プリローンチで初動の注文数を確保する意義

2. レビュー数 × 評価
├── レビュー数が多い商品は信頼性が高いと判断
├── 平均評価4.0以上を維持（3.5未満は致命的）
├── 直近のレビューが重視される
└── Vineプログラム・フォローアップで獲得

3. CTR（クリック率）
├── 検索結果での商品のクリック率
├── メイン画像の品質が直結する
├── タイトルの訴求力
├── レビュー星評価の表示
└── 価格の競争力

4. CVR（転換率）
├── 商品ページ訪問者の購入率
├── 画像・テキスト・A+の品質
├── 価格の妥当性
├── レビューの質
└── 在庫の安定性（品切れはCVRに致命的）

5. セッション数
├── 商品ページの訪問数
├── 広告 + オーガニック検索 + 外部流入
├── セッション数自体もシグナルとなる
└── プリローンチの外部流入も貢献
```

### SEO改善サイクル
```
週次SEOレビューの手順:

Step 1: キーワードランキングの確認
├── 主要KW（5-10個）の順位を追跡
├── 先週比での変動を確認
└── 競合との順位比較

Step 2: セッション・CTR・CVRの確認
├── セラーセントラルのビジネスレポート
├── 検索語句レポート（広告経由）
└── 異常値の有無をチェック

Step 3: 改善アクションの決定
├── 順位下落KW → 広告入札強化 or テキスト見直し
├── CTR低下 → メイン画像の変更テスト
├── CVR低下 → 価格・画像・レビュー状況の確認
└── セッション減少 → 広告費の見直し or 外部流入強化

Step 4: テキストSEOの更新（必要に応じて）
├── 新規キーワードの追加（バックエンド）
├── タイトルの微調整
├── 箇条書きの訴求力強化
└── A+コンテンツの更新
```

## オーガニック比率の目標

```
健全なSEO状態の指標 = オーガニック売上比率

オーガニック比率 = オーガニック売上 / 総売上 × 100

フェーズ別目標:
├── 発売1ヶ月目: 10-20%（ほぼ広告依存は正常）
├── 発売2ヶ月目: 30-40%
├── 発売3ヶ月目: 50-60%
├── 発売4ヶ月目以降: 70%以上（目標）
└── 安定期: 70-80%（広告は20-30%で維持）

★ オーガニック比率70%以上 = SEO成功の証
★ 広告を止めても売上の70%は維持できる状態
★ この状態になったら広告費を段階的に最適化
```

## テキストSEO × 実績SEOの相乗効果

```
テキストSEO × 実績SEOの正のスパイラル:

テキストSEOで「保温マグカップ」にインデックス
  ↓
広告で「保温マグカップ」の検索結果に表示
  ↓
クリック → 購入（注文数UP）
  ↓
実績SEOスコア向上
  ↓
オーガニック検索順位UP
  ↓
広告なしでもクリック → 購入
  ↓
さらに実績SEOスコア向上（正のスパイラル）

★ テキストSEOが不十分 → そもそも表示されない → 広告でもCTRが低い
★ 実績SEOが不十分 → 表示されるが順位が低い → クリックされない
★ 両方を同時に攻略することで加速度的に成長する
```

## キーワード戦略

### キーワードの階層構造
```
キーワードの3階層:

Tier 1: ビッグキーワード（月間検索数 10,000+）
├── 例: 「マグカップ」「水筒」「イヤホン」
├── 上位表示難易度: ★★★★★
├── 狙い方: 長期的にSEOで上位を目指す
└── 広告: 入札が高い。初期は避けてもOK

Tier 2: ミドルキーワード（月間検索数 1,000-10,000）
├── 例: 「保温マグカップ 蓋付き」「ワイヤレスイヤホン 防水」
├── 上位表示難易度: ★★★
├── 狙い方: メインターゲット。タイトルに含める
└── 広告: メインの広告ターゲティング

Tier 3: ロングテールキーワード（月間検索数 100-1,000）
├── 例: 「保温マグカップ オフィス 500ml ステンレス」
├── 上位表示難易度: ★
├── 狙い方: 箇条書き・バックエンドに含める
└── 広告: CPC安い。高CVR。初期のメインターゲット

戦略:
├── 初期: Tier 3で確実に上位 → 実績を蓄積
├── 中期: Tier 2で上位を狙う → 広告を活用
├── 長期: Tier 1で上位を目指す → オーガニック中心
└── ★ 下から積み上げるのが正攻法
```
