# 商品カタログ作成 詳細ガイド（Amazon自社ブランド戦略フレームワーク）

## カタログ作成の重要性

```
Amazon商品カタログ = テキストSEOの基盤 + CVRの決定要因

カタログの品質が決めること:
├── テキストSEO: どのキーワードで検索結果に表示されるか
├── CTR: 検索結果でクリックされるか（メイン画像+タイトル）
├── CVR: 商品ページで購入されるか（画像+テキスト+A+）
└── ★ カタログが悪いと広告をいくらかけても売れない
```

## 商品画像（7枚構成）

```
Amazon商品画像の戦略的構成:

画像1: メイン画像（白背景）
├── Amazon規約: 純白背景（RGB 255,255,255）
├── 商品が画像の85%以上を占めること
├── テキスト・ロゴ・ウォーターマーク禁止
├── ★ 検索結果でのCTRを決める最重要画像
└── プロカメラマンによる撮影を推奨

画像2: ベネフィット訴求（インフォグラフィック）
├── 最大のベネフィットを視覚的に表現
├── テキストオーバーレイで特徴を説明
├── 例: 「6時間保温」「防水IPX7」等の数値訴求
└── ★ 画像を見ただけで商品の価値がわかるように

画像3: 特徴説明（3-5ポイント）
├── 商品の主要特徴をアイコン+テキストで列挙
├── 素材・機能・サイズ等の重要情報
└── ★ テキストは大きく、スマホでも読めるサイズ

画像4: 使用シーン（ライフスタイル）
├── 実際の使用場面を見せる
├── ターゲット層が共感するシチュエーション
├── 例: オフィス使用、アウトドア、家族での使用
└── ★ 「自分が使うイメージ」を持たせる

画像5: サイズ・寸法
├── 商品の寸法を図示
├── 手や日用品との比較（スケール感）
├── 重量情報
└── ★ 返品理由「想像と違った」を減らす

画像6: 素材・品質の訴求
├── 素材のクローズアップ
├── 製造工程・品質管理の紹介
├── 認証・検査結果
└── ★ 安心感・信頼感の醸成

画像7: パッケージ/付属品/保証
├── 箱・パッケージの外観
├── 付属品一覧
├── 保証情報・サポート連絡先
└── ★ 「何が届くか」を明確にして不安を解消

画像制作のルール:
├── 解像度: 最低1000×1000ピクセル（推奨2000×2000）
├── ファイル形式: JPEG/PNG
├── テキスト: 日本語で読みやすく、フォントサイズ大きめ
├── モバイル確認: スマホで見ても文字が読めるか必ずチェック
└── 競合分析: TOP5商品の画像構成を参考に差別化
```

## タイトル最適化

```
タイトル設計のフレームワーク:

構成: [ブランド名] [メインKW] [サブKW] [特徴] [スペック]
文字数: 全角60-80文字（スマホ表示は先頭40文字が特に重要）

キーワード選定手順:
├── Step 1: Helium10/Jungle Scoutでメインキーワード特定
│   └── 月間検索ボリューム最大のKW
├── Step 2: サブキーワード2-3個を選定
│   └── メインKWと異なる検索意図のKW
├── Step 3: 特徴・スペックで補完
│   └── サイズ・色・素材等の絞り込みKW
└── Step 4: 自然な日本語になるよう構成

例:
├── ✅ 「[ブランド名] 保温マグカップ 真空断熱 500ml ステンレス製 蓋付き オフィス アウトドア」
├── ❌ 「マグカップ 保温 真空 断熱 ステンレス 蓋 オフィス キャンプ コーヒー 紅茶」（KW羅列）
└── ❌ 「【最高品質】究極の保温マグカップ！売上No.1！」（主観的表現・虚偽表示）
```

## 箇条書き（Bullet Points）

```
5つの箇条書き設計テンプレート:

Point 1: 【メインベネフィット】+ メインKW
├── 商品の最大の価値を訴求
├── 例: 「【6時間保温で冷めない】真空断熱構造により、朝入れたコーヒーがランチタイムまで温かい」
└── KWを自然に含める

Point 2: 【差別化ポイント】+ サブKW
├── 競合にない特徴を訴求
├── 例: 「【片手で開閉できるワンタッチ蓋】忙しいデスクワーク中でも片手で簡単操作」
└── 競合レビューの不満点を解決している旨を示す

Point 3: 【素材・品質】
├── 安心感を与える情報
├── 例: 「【SUS304ステンレス採用】食品衛生法適合、BPAフリーで安心安全」
└── 認証・検査情報があれば記載

Point 4: 【用途・シーン提案】
├── 使用場面を具体的にイメージさせる
├── 例: 「【オフィス・車内・アウトドアに】底面ゴムパッド付きでデスクでも滑りにくい」
└── ギフト用途も訴求（該当する場合）

Point 5: 【保証・サポート】
├── 購入の不安を解消
├── 例: 「【30日間返金保証】万が一ご満足いただけない場合、全額返金対応いたします」
└── 日本語サポートの連絡先

書き方のルール:
├── 各ポイントの冒頭に【見出し】を入れる（視認性UP）
├── ベネフィット中心（○○だから□□できる）
├── 数字を使って具体的に（「約6時間」より「6時間」）
├── 1ポイントあたり200-250バイト以内
└── ★ スペック羅列ではなく「お客様にとっての価値」を伝える
```

## A+コンテンツ

```
A+コンテンツの構成設計:

前提: ブランド登録（Brand Registry）完了済み

推奨モジュール構成:

Module 1: ブランドストーリーヘッダー
├── ブランドロゴ
├── ブランドの理念・背景を簡潔に
└── 信頼性の構築

Module 2: メインベネフィット（大型画像+テキスト）
├── 最大の特徴を大きな画像で訴求
├── ライフスタイル画像 + 説明テキスト
└── 画像のaltテキストにKWを含める

Module 3: 3つの特徴（3列レイアウト）
├── アイコン/画像 + テキストの3列構成
├── 差別化ポイントを3つピックアップ
└── 各特徴を簡潔に（2-3行）

Module 4: 使用方法/活用シーン
├── Step-by-step の使い方
├── 複数の活用シーン提案
└── 「こんな方におすすめ」のターゲット明示

Module 5: スペック/仕様表
├── 詳細な製品仕様
├── サイズ・重量・素材・同梱物
└── 比較表（自社商品ラインがある場合）

Module 6: お客様の声/レビュー紹介
├── 代表的な高評価レビューの紹介（許可を得て）
├── 使用者の写真（モニター画像等）
└── ★ 社会的証明でCVR向上

Module 7: FAQ / サポート情報
├── よくある質問と回答
├── 保証情報
├── 問い合わせ先
└── 返品・交換ポリシー

制作の注意点:
├── テキストにもKWを自然に含める（インデックス対象）
├── 画像はモバイルで確認（表示が小さくなる）
├── 定期的に更新（レビューの声を反映）
├── 競合のA+コンテンツを分析して差別化
└── ★ A+コンテンツの有無でCVRが3-10%変わるという報告あり
```

## バックエンドキーワード設定

```
バックエンドキーワード設定ガイド:

設定場所: セラーセントラル → 在庫 → 商品情報の編集 → キーワード

文字数: 500バイト以内（日本語約166文字）

含めるべきキーワード:
├── タイトルに入らなかった関連KW
├── 類義語・別表現
│   ├── 例: 「マグカップ」→「コップ」「カップ」「タンブラー」
│   └── ひらがな/カタカナ表記揺れ
├── 用途・シーン
│   ├── 「プレゼント」「ギフト」「誕生日」
│   └── 「オフィス」「アウトドア」「車」
├── 素材・特徴の別表現
│   └── 「ステンレス」「真空」「二重構造」
└── 季節・イベント（該当する場合）

含めてはいけないもの:
├── ❌ 競合ブランド名
├── ❌ ASIN
├── ❌ 「最安」「激安」「No.1」等
├── ❌ すでにタイトルに含まれているKW
└── ❌ 無関係なKW

設定テクニック:
├── スペース区切り（カンマ不要）
├── 重複排除（同じKWを2回入れても意味なし）
├── 月1回の見直し（検索トレンドの変化に対応）
└── 新しいKWは検索語句レポートから発見
```

## カタログ改善サイクル

```
カタログは「作って終わり」ではない。継続的に改善する:

月次レビュー項目:
├── CTR（検索結果でのクリック率）
│   ├── 低い場合 → メイン画像の変更テスト
│   └── 低い場合 → タイトルの訴求力を見直し
│
├── CVR（商品ページでの転換率）
│   ├── 低い場合 → サブ画像の改善
│   ├── 低い場合 → 箇条書きの訴求力を見直し
│   ├── 低い場合 → 価格の見直し
│   └── 低い場合 → A+コンテンツの改善
│
├── 検索順位の変動
│   ├── 下落KW → バックエンドKWの見直し
│   └── 新規KW追加の検討
│
└── レビューの内容
    ├── 新しい不満点 → 商品改善 or カタログで説明強化
    └── 好評ポイント → カタログでさらに訴求

ABテストの進め方:
├── Amazon Manage Your Experiments（MYE）を活用
├── テスト対象: メイン画像 / タイトル / A+コンテンツ
├── テスト期間: 最低2週間（4週間推奨）
├── 判定: 統計的有意性が出たら勝者を採用
└── ★ 一度に1要素のみテスト（複数変更すると効果が不明）
```
