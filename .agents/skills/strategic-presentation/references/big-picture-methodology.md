# 「大きな絵」メソドロジー詳細

## 出典・インスピレーション

本メソッドは、アクセンチュア シニアマネージャー出身の鴨居啓人氏による
「たった１枚のスライドで数億円の案件が決まる」アプローチに基づく。

## 核心思想

### 1. スライドは「コミュニケーション」である

```
スライドの本質:
├── 単なる「情報の羅列」ではない
├── 相手に「伝える」ためのツール
├── 美しいスライド = 相手への思いやり
└── 相手が理解し、行動するために存在する
```

### 2. 「1枚のスライド + 3つのキーワード」の制約

```
なぜ制約が重要か:
├── 経営層の時間は限られている
├── 情報が多いほど、伝わらない
├── 本質を絞り込む努力が説得力を生む
└── 「削ぎ落とす」ことで核心が見える

3つのキーワードの力:
├── 人間が一度に記憶できるのは3-5項目
├── 3つは「少なすぎず、多すぎない」最適数
├── 各キーワードが独立して成立する
└── 合わせて全体をカバーする（MECE）
```

### 3. 視座を上げる技術

```
担当者レベル:
「LP改善で直帰率を下げたい」
    ↓
部門長レベル:
「EC事業の売上を向上させたい」
    ↓
経営レベル:
「会社の成長戦略を実現したい」

視座を上げる = より大きな課題に接続する
```

## 実践フレームワーク

### Step 1: 依頼の「背後」を読む

```
質問リスト:
1. この依頼者の上司は誰か？
2. その上司は何を求めているか？
3. この課題が解決されると、会社全体にどう影響するか？
4. なぜ「今」この依頼が来たのか？
5. 競合がこれを先にやったら、どうなるか？

実践例:
依頼: 「万能LPの直帰率を改善したい」
背後を読む:
├── 上司: EC事業責任者
├── 上司の関心: 売上目標の達成
├── 会社への影響: 主力商品の成長
├── タイミング: 競合が台頭している
└── 再定義: 「EC売上2倍への成長戦略」
```

### Step 2: 「大きな絵」を描く

```
小さな依頼 → 大きな絵への変換テンプレート:

┌─────────────────────────────────────────┐
│ 依頼: [具体的な作業依頼]               │
│ 例: 「LP直帰率改善」                   │
└─────────────────────────────────────────┘
            ↓
┌─────────────────────────────────────────┐
│ 背後の課題: [より大きな課題]           │
│ 例: 「EC売上の伸び悩み」               │
└─────────────────────────────────────────┘
            ↓
┌─────────────────────────────────────────┐
│ 大きな絵: [経営インパクトで再定義]     │
│ 例: 「EC売上2倍への成長戦略」          │
└─────────────────────────────────────────┘
            ↓
┌─────────────────────────────────────────┐
│ 3つの柱: [実現方法を3つに整理]         │
│ 例:                                     │
│ 1. 導線最適化（LP→販売ページ）         │
│ 2. セグメント戦略（新規/リピーター）   │
│ 3. 継続改善（データドリブンPDCA）      │
└─────────────────────────────────────────┘
```

### Step 3: 「覚悟」を込める

```
大きな絵を描く = 大きな覚悟を示す

鴨居氏の言葉:
「大きな提案は、自分がその会社の成功に
 100％責任を持つ、という覚悟の表現」

覚悟の表現方法:
├── 具体的な数字でコミットする
├── 実行計画を詳細に示す
├── リスクと対策を明示する
├── Next Steps で即行動を示す
└── 自分も巻き込まれる覚悟を見せる
```

## Why → What → How の詳細

### WHY（なぜ）

```
Why You?（なぜあなたか）:
├── 現状の強み
│   └── 「商品力は証明済み（直接流入CVR 3.67%）」
├── 改善余地の明確さ
│   └── 「万能LP遷移率25% vs 米びつLP 39%」
└── 改善の確実性
    └── 「同じ商品で既に成功パターンがある」

Why Now?（なぜ今か）:
├── 機会損失の可視化
│   └── 「毎月¥391万の機会損失が発生中」
├── 時間的優位性
│   └── 「競合が改善を進める前に」
└── 投資済みリソースの活用
    └── 「広告費は投下済み、活かしきれていない」
```

### WHAT（何を）

```
3つの柱の設計原則:
├── 相互に重複しない（MECE）
├── 合わせて全体をカバーする
├── 各柱が独立して成立する
├── 優先順位がある（左から右）
└── 数字で効果を示せる

例:
柱1: 導線最適化
├── 内容: LP→販売ページの遷移率を1.5倍に
├── 施策: ファーストビュー改善、CTA最適化
└── 効果: +¥391万/月

柱2: セグメント戦略
├── 内容: 新規/リピーター別に最適LP出し分け
├── 施策: Meta広告でカスタムオーディエンス
└── 効果: 即実行可能、低コスト

柱3: 継続改善
├── 内容: データドリブンPDCA確立
├── 施策: ヒートマップ分析、A/Bテスト
└── 効果: 持続的成長
```

### HOW（どうやって）

```
実行ロードマップの設計:

Phase 1（即効性施策）:
├── 今すぐ始められる
├── 低コスト or コストゼロ
├── 効果検証が早い
└── 小さな成功体験を得る

Phase 2（本格施策）:
├── Phase 1の成果を踏まえて
├── 投資が必要
├── 効果が大きい
└── 成功パターンを拡大

Phase 3（継続施策）:
├── システム化・自動化
├── 組織能力として定着
├── 持続的な成長エンジン
└── 次のフェーズへの布石
```

## 成功事例

### 事例: ECサイト優良顧客分析

```
【依頼】
「優良顧客分析をしたい」
（見積もり: 500万〜1000万円規模）

【再定義】
「顧客エンゲージメント向上とEC売上増」
├── 優良顧客分析は「入り口」に過ぎない
├── 本当の課題はEC全体の成長
└── 大規模リニューアルの提案へ

【結果】
数億円規模のプロジェクトへ成長
```

### 事例: LP改善提案

```
【依頼】
「万能LPの直帰率を改善したい」
（見積もり: 50万〜100万円規模）

【再定義】
「EC売上2倍への成長戦略」
├── LP改善は「手段の一つ」
├── 本当の課題は売上成長
├── 3つの柱で全体戦略を提示
└── 年間+¥4,700万〜¥8,160万のインパクト

【3つのキーワード】
1. 導線最適化
2. セグメント戦略
3. 継続改善
```

## よくある失敗パターン

### 失敗1: 依頼をそのまま資料化

```
❌ 悪い例:
「LP直帰率改善のご提案」
├── 現状の直帰率
├── 改善施策
└── 以上

問題点:
├── 視座が上がっていない
├── 経営インパクトが見えない
├── 「だから何？」になる
└── 意思決定につながらない
```

### 失敗2: キースライドがない

```
❌ 悪い例:
スライド1: 背景
スライド2: 現状分析
スライド3: 課題整理
...
スライド15: 結論

問題点:
├── 最後まで見ないと結論がわからない
├── 経営層は最初の数枚で判断する
├── 時間がないと途中で終わる
└── 記憶に残らない
```

### 失敗3: 数字がない

```
❌ 悪い例:
「改善の余地があります」
「効果が期待できます」
「やる価値があると思います」

問題点:
├── 定量的な説得力がない
├── 比較ができない
├── 意思決定の根拠にならない
└── 「思います」は信頼を失う
```
