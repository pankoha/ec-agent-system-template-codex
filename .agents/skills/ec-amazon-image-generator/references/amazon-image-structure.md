# Amazon画像構成リファレンス

## 基本思想

Amazonの商品画像は、単なる商品説明ではなく「クリックから購入までの不安解消シナリオ」として設計する。

購買導線:

1. 気づく: メイン画像で商品を正確かつ魅力的に認識する
2. 興味を持つ: ベネフィット画像で自分ごと化する
3. 理解する: 機能・特徴画像で価値を把握する
4. 想像する: 使用シーン画像で購入後をイメージする
5. 比較する: 比較・証明画像で選ぶ理由を得る
6. 安心する: サイズ・使い方・保証・同梱物で不安を消す
7. 決める: クロージング画像で購入理由を再確認する

## 1. メイン画像

目的: 検索結果と商品ページ初見でクリックを取る。

必須:
- 白背景
- 販売商品だけを表示
- 商品が大きく、全体が見える
- 輪郭、素材、色が正確
- 画像の主役が一瞬でわかる

禁止:
- テキスト、ロゴ、バッジ、枠、装飾
- 商品に含まれない小物や背景
- 生活シーン、モデル、手持ち表現
- 誤認を招くセット数や付属品

推奨指示:
- 商品占有率を高くする
- 自然な影で接地感を出す
- レタッチで傷、埃、色ムラを整える
- 反射、光沢、エッジを整えて高品質に見せる

## 2. ベネフィット訴求画像

目的: 「これは自分の悩みを解決する商品だ」と直感させる。

構成:
- 大きなキャッチコピー
- 悩み、または理想状態
- 商品写真または使用結果イメージ
- 1つの主要ベネフィット

コピー例の型:
- 「もう、○○で悩まない」
- 「毎日の○○をもっと快適に」
- 「○○するだけで、△△へ」
- 「その不満、○○で解決」

注意:
- 断定しすぎない
- 薬機法・医療効果・身体変化の表現に注意
- Before/Afterは根拠と表現規制を確認

## 3. 機能・特徴画像

目的: 商品の強みを短時間で理解させる。

構成:
- 3-5個の特徴
- アイコン、丸バッジ、吹き出し、拡大カット
- 機能とベネフィットをセットで表示

変換例:
- 「高密度素材」→「へたりにくく長く使える」
- 「600ml」→「外出先でも1日分をしっかり持ち運べる」
- 「防水IPX7」→「雨の日や水まわりでも使いやすい」

## 4. 使用シーン画像

目的: 購入後の生活を想像させる。

構成:
- ターゲットに近い人物・場所・時間帯
- 商品を自然に使っている瞬間
- 背景は商品ジャンルと生活文脈に合わせる

注意:
- メイン画像では使わない
- モデル生成は権利、肖像、年齢、身体表現に注意
- 商品のサイズや使い方が誤解されないようにする

## 5. 比較・証明画像

目的: 競合ではなくこの商品を選ぶ理由を示す。

構成:
- 比較表、チェックリスト、従来品比較
- 数値、素材、容量、耐久性、保証などの根拠
- 自社優位点を3-5個に絞る

注意:
- 競合名を出す場合は根拠と法務確認が必要
- 根拠のないNo.1、最高、最安、圧倒的などは避ける
- 「一般品」「従来品」表現も実態が必要

## 6. サイズ・同梱物・使い方画像

目的: 購入前の不安を潰す。

構成:
- 寸法線、重量、容量、同梱物一覧
- 人の手、家具、バッグなどサイズ比較
- 3ステップ程度の使い方

注意:
- メイン画像では商品に含まれない比較物を入れない
- 同梱物は正確に表示する

## 7. クロージング画像

目的: 購入理由を再確認させる。

構成:
- 主要ベネフィットの再整理
- 保証、サポート、ブランドの信頼
- 使用後の理想状態

注意:
- 過剰な煽りを避ける
- レビュー風表現や実績は根拠がある場合のみ使う

## Aプラスコンテンツ

目的: 商品ページ下部で、ブランド世界観と購入納得感を補強する。

推奨モジュール:
- ブランド導入
- 商品の主要ベネフィット
- 機能詳細
- 使用シーン
- 比較表
- よくある疑問

設計原則:
- 商品画像より情報量を増やしてよい
- モバイル表示で読みやすい文字量にする
- 画像単体で意味が通るようにする
- ブランドトーンを商品画像セットと統一する
