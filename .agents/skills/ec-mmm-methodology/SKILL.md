# 📊 MMM METHODOLOGY（マーケティング・ミックス・モデリング方法論）

## 役割
全マーケティング関連Agentが共有するMMM分析の知識基盤。
MMMエンジンの出力を正しく解釈し、科学的根拠に基づく意思決定を支援する。

## 対象Agent（COMMANDERが選定）
| Agent | MMMスキルの活用場面 |
|-------|-------------------|
| **ec-insight** | 結果解釈・レポーティング・異常検知・精度評価 |
| **ec-acquisition** | 飽和曲線に基づくチャネル別予算配分・入札戦略 |
| **ec-commander** | 全社予算配分の戦略的判断・フェーズ別投資方針 |
| **ec-operations** | プラットフォームチャネルの売上構成比分析・CVR連動 |
| **cfo** | ROI/CPA の財務的妥当性評価・投資判断 |

## フェーズ別MMMの活用重点
| Phase | MMM活用の重点 |
|-------|-------------|
| Phase 0 | ベースライン構築。最低60日分のデータで初回MMM実行。チャネル特性の把握 |
| Phase 1 | 勝ちチャネル特定。飽和度の低いチャネルへ集中投資。限界ROIベースの配分開始 |
| Phase 2 | チャネル拡大。シナジー分析でチャネル間相乗効果を特定。予算配分の最適化本格運用 |
| Phase 3 | 効率最大化。過飽和チャネルの予算削減。限界CPA基準での全チャネル最適化 |
| Phase 4 | 利益最大化。プラットフォームチャネルのコスト構造分析。全チャネルの限界ROI均衡化 |

## Skills

### Skill 1: MMMモデルの理解（Model Architecture）
Robyn準拠Ridge回帰MMMの構造を正確に理解し、各コンポーネントの意味を説明できる。

**パイプライン構造:**
```
日次チャネル別支出データ + 外部要因
  ↓
Step 1: アドストック変換（Geometric Adstock）
  └─ adstocked[t] = spend[t] + decay × adstocked[t-1]
  └─ チャネル別減衰率で広告効果の残存を反映
  ↓
Step 2: Hill飽和曲線フィッティング（チャネル別）
  └─ revenue = max_revenue × spend^S / (K^S + spend^S)
  └─ K = 半飽和点、S = 曲線の急峻度
  ↓
Step 3: Ridge回帰（多変量）
  └─ Y(t) = 切片 + Σβ_j×Hill(Adstock(spend_j(t))) + 曜日効果 + 外部要因
  ↓
Step 4: チャネル分解・予算最適化・シナジー分析
```

**アドストック減衰率（デフォルト）:**
| チャネル | 減衰率 | 意味 |
|---------|--------|------|
| Google Ads | 0.5 | 検索+ディスプレイの中程度残存 |
| Meta Ads | 0.4 | 認知系で比較的長い残存 |
| TikTok | 0.2 | 短期的インパクト |
| Amazon | 0.15 | 検索意図直結、ほぼ即時効果 |
| 楽天 | 0.2 | モール内検索広告 |
| SEO | 0.7 | オーガニック、最も長い残存効果 |
| Email | 0.15 | ダイレクトレスポンス |

> 詳細は `references/model-architecture.md` を参照

### Skill 2: モデル精度の評価（Model Quality Assessment）
R²とMAPEの正しい解釈基準を持ち、モデルの信頼性を判断できる。

**R²（決定係数）の判断基準:**
| R² | 判定 | アクション |
|----|------|----------|
| 0.85+ | 優秀 | 結果を高い信頼度で活用可能 |
| 0.70-0.85 | 良好 | 方向性の判断に活用。細かい数値は参考程度 |
| 0.50-0.70 | 注意 | 大きなトレンドのみ参考。チャネル別の細かい比較は危険 |
| 0.50未満 | 不十分 | データ品質の見直しが必要。結果を意思決定に使わない |

**MAPE（平均絶対誤差率）の判断基準:**
| MAPE | 判定 | 意味 |
|------|------|------|
| 15%未満 | 優秀 | 日次予測としても高精度 |
| 15-25% | 良好 | 週次・月次の予算配分判断に十分 |
| 25-40% | 許容 | 大まかな方向性のみ。チャネル間の相対比較は可能 |
| 40%超 | 不十分 | データ期間の延長またはデータ品質改善が必要 |

**精度が低い場合のチェックポイント:**
```
□ データ期間は90日以上あるか？（最低60日、推奨180日+）
□ 欠損日がないか？（歯抜けデータはモデル精度を著しく低下させる）
□ チャネルの支出パターンに十分なバリエーションがあるか？
□ 大型セール等の外部要因が登録されているか？
□ プラットフォームチャネルと広告チャネルが正しく分類されているか？
```

### Skill 3: 飽和曲線の読み方と投資判断（Saturation Curve Interpretation）
飽和度・Hill K/Sパラメータから各チャネルの投資余地を正確に判断できる。

**飽和度別アクション基準:**
| 飽和度 | 状態 | アクション |
|--------|------|----------|
| 0-30% | 大幅な投資余地 | 積極投資推奨。限界ROIが高い黄金ゾーン |
| 30-60% | 適度な投資余地 | 段階的に増額。ROI変化をモニタリング |
| 60-80% | 飽和傾向 | 増額は慎重に。他チャネルとのバランスを優先 |
| 80-95% | 過飽和間近 | 現状維持または微減。追加投資の効果は限定的 |
| 95%+ | 過飽和 | 予算削減を検討。他チャネルへの再配分を推奨 |

**Hill パラメータの解釈:**
```
K（半飽和点）:
├── 小さいK = 少額で効果が出始める（ニッチチャネル、検索意図型）
└── 大きいK = 一定の投資量がないと効果が出にくい（認知型、ブロードリーチ）

S（曲線の急峻度）:
├── S > 2 = 急激な飽和（閾値を超えると一気に効果が頭打ち）
├── S ≈ 1 = 緩やかな飽和（線形に近い応答）
└── S < 1 = 非常に緩やか（投資量に比例して徐々に効果逓減）

Hill R²（フィット精度）:
├── 0.5+ = 良好なフィット。飽和曲線の形状を信頼してよい
├── 0.1-0.5 = 参考程度。大まかな傾向として活用
└── 0.1未満 = フィット不良。飽和曲線の数値を意思決定に使わない
```

> 詳細は `references/saturation-analysis.md` を参照

### Skill 4: 限界ROI vs 平均ROI の区別（Marginal vs Average ROI）
マーケティング投資判断で最も重要な「限界」の概念を正しく理解・適用する。

**致命的な誤解を防ぐ:**
```
❌ 「ROASが6倍だから、もっと投資すべき」
   → 平均ROASは過去の蓄積。次の1円の効果（限界ROI）は別の話

❌ 「CPAが安いから効率が良い」
   → 平均CPAが安くても、追加投資の限界CPAは高い可能性がある

✅ 「限界ROIが2.5倍なので、まだ追加投資の余地がある」
   → 次の1円を投資した時のリターンで判断

✅ 「限界CPAが¥15,000を超えたので、このチャネルの増額は止めるべき」
   → 目標CPA基準で限界効率を評価
```

**指標の関係性:**
```
平均ROI = 総売上 / 総コスト（期間全体の効率）
限界ROI = ΔRevenue / ΔSpend（追加1円あたりの効果）
平均CPA = 総コスト / 総注文数
限界CPA = 1 / 限界ROI × 平均単価（追加1件あたりのコスト）
True CPA = アドストック考慮コスト / 注文数（残存効果を含む真のCPA）

投資判断の原則:
├── 限界ROI > 1.0 → 追加投資で利益が出る
├── 限界ROI = 1.0 → 損益分岐点
└── 限界ROI < 1.0 → 追加投資は赤字（削減すべき）
```

### Skill 5: チャネル間シナジー分析（Cross-Channel Synergy）
チャネル間の相関から相乗効果を特定し、統合的なメディアプランに活かす。

**相関係数の解釈:**
| 相関 | 意味 | アクション |
|------|------|----------|
| 0.7+ | 高い正の相関 | 同時投資で相乗効果の可能性。片方を削ると両方に影響 |
| 0.4-0.7 | 中程度の相関 | シナジーの可能性あり。A/Bテストで因果関係を検証 |
| -0.2-0.4 | 弱い/無相関 | 独立したチャネル。個別に最適化可能 |
| -0.2未満 | 負の相関 | 予算のトレードオフ関係。配分バランスに注意 |

**シナジー分析の活用例:**
```
Amazon ↔ 楽天: 相関 0.70
→ モール広告を同時に強化すると相乗効果が期待できる
→ 片方だけ削減するとリスク。両方セットで検討

Shopify ↔ Meta Ads: 相関 0.67
→ Meta広告が自社ECの売上ドライバー。Meta削減は自社EC直撃
→ 自社ECの売上目標とMeta予算を連動させる

Google Ads ↔ 全チャネル: 無相関
→ 独立したチャネル。Google Ads単体で最適化判断可能
→ 他チャネルの予算変更の影響を受けにくい
```

### Skill 6: 予算最適化の実践（Budget Optimization）
MMMエンジンの予算最適化出力を正しく解釈し、実行可能な予算計画に落とし込む。

**最適化の仕組み:**
```
目的: 全広告チャネルの限界ROIを均衡化する
方法: 限界ROIの高いチャネルに予算を移動、低いチャネルから削減
制約: 上限 3.0倍 / 下限 0.1倍（急激な変更を防止）

プラットフォームチャネル（Shopify/BASE等）:
→ 最適化対象外（手数料ベースで直接コントロール不可）
```

**推奨予算の読み方:**
```
Current vs Recommended の差が意味すること:
├── +200%（上限キャップ） → 投資余地が非常に大きい。段階的に増額を検討
├── +50%〜100% → 明確な増額推奨。限界ROIが高い
├── -10%〜+10% → ほぼ適正。現状維持
├── -10%〜-30% → 軽度の過剰投資。段階的に削減
└── 上限0.1×にキャップ → 限界ROIが極めて低い。大幅削減を検討

capped = "upper" の場合:
→ 制約がなければもっと増やすべき。段階的に上限を緩和して再分析

capped = "lower" の場合:
→ 制約がなければもっと減らすべき。本当に必要か根本から再検討

marginal_roi = 0.0 の場合:
→ そのチャネルの追加投資は実質無効。100%過飽和状態
```

> 詳細は `references/budget-optimization.md` を参照

### Skill 7: 外部要因とウィンドウ分析（External Factors & Window Analysis）
PR・TV・イベント等の外部要因が広告効果に与えた影響を定量的に評価する。

**ウィンドウ分析の読み方:**
```
各外部イベントについて:
  前期間（7日間）vs 後期間（7日間）のROAS・売上を比較

  revenue_lift_pct > 0 → イベント後に売上向上（正の効果）
  revenue_lift_pct < 0 → イベント後に売上低下（負の効果またはタイミング要因）
  is_significant = true → 統計的に有意（p < 0.05）
  is_significant = false → 偶然の可能性あり。サンプル不足の場合も

活用法:
├── 有意な正の効果 → 同種イベントの再実施を計画。広告と連動させる
├── 有意な負の効果 → イベント内容・タイミングの見直し
└── 非有意 → データ不足か効果なし。追加検証が必要
```

**コンテキスト変数（Ridge回帰の係数）:**
```
coefficient > 0 → その外部要因は売上にプラスの影響
coefficient < 0 → その外部要因は売上にマイナスの影響
occurrences = 1 → 単発イベント（統計的信頼性は低い）
occurrences = 5+ → 繰り返しイベント（信頼性が高い）
```

## 解像度プロトコル — MMM分析の深掘り（全Agent必須）

### MMM結果のWhy So?ループ
```
MMM結果を見て「チャネルAのROASが低い→予算削減」と即断してはならない。
必ずWhy So?で根本原因を特定してからアクションを決定する。

例）「Meta AdsのROASが1.77倍で低い」
├── Why So? → 飽和度69.1%で飽和傾向にある
├── Why So? → 月額¥1,800万の高額投資で限界効率が低下
├── Why So? → オーディエンスの拡大余地がなく、同じ層に繰り返し配信
├── Why So? → クリエイティブの疲弊。3ヶ月以上同じ訴求を使用
└── Why So? → CREATIVEリソースが既存運用に取られ、新規テスト素材が不足
→ 根本原因: 「予算削減」ではなく「クリエイティブリフレッシュ + オーディエンス拡張」

❌ 禁止: 「ROASが低い→予算削減」（1段階の因果で結論）
❌ 禁止: 「飽和度が高い→もう伸びない」（原因を掘り下げていない）
✅ 必須: 最低5回のWhy So?で、数値の裏にある構造的原因を特定
✅ 必須: 根本原因に対するアクションを設計してから予算を動かす
```

### MMM結果の解像度チェック
```
以下に該当する場合、分析の解像度が低いと判断し再分析すること:

⚠️ 「全チャネル増額推奨」→ 総予算制約を無視している。トレードオフを考慮
⚠️ 「ROASだけで判断」→ 限界ROI、飽和度、シナジーを含めた総合判断が必要
⚠️ 「モデル精度に言及しない」→ R²/MAPEを必ず記載し、信頼度を明示
⚠️ 「前回と比較しない」→ 前回MMM結果との差分がなければ変化を捉えられない
⚠️ 「プラットフォーム費用を無視」→ Shopify手数料等も含めた真のコスト構造で評価
⚠️ 「曜日効果に触れない」→ 曜日による売上変動を把握していない
```

### N=1チャネル深掘り分析
```
MMM結果からアクションを決める前に、最も重要な1チャネルを深掘りする。

対象チャネルの選定基準:
├── 売上構成比が最大のチャネル（ビジネスインパクト最大）
├── 飽和度の変化が最も大きいチャネル（急激な変化は要注意）
└── 限界ROIと平均ROIの乖離が最も大きいチャネル（非効率の可能性）

深掘り項目:
├── 過去3回のMMM結果での飽和度推移は？（トレンドを把握）
├── 日次データで見た時、効果が出ている日と出ていない日の違いは？
├── アドストック減衰率は妥当か？（デフォルト値が合っていない可能性）
├── Hill R²が低い場合、データの分布に偏りがないか？
└── 外部要因（セール、PR）の影響を除外するとどうなるか？
```

## Review & Challenge（壁打ち・批判的検証）

MMM結果に基づく提案は、以下の6つの検証レンズで必ず批判的にレビューする。

### MMM固有の検証レンズ
```
① モデル信頼性: R²/MAPEは十分か？Hill R²が低いチャネルの結論を信じていないか？
② 限界 vs 平均: 平均ROASだけで判断していないか？限界ROIを確認したか？
③ 飽和度の構造: 飽和の原因を特定したか？クリエイティブ/オーディエンス/季節性？
④ シナジー考慮: チャネル単体ではなく、相関するチャネルへの波及効果を考えたか？
⑤ 実行可能性: 推奨予算変更を実際に実行できるか？最低出稿額・契約制約は？
⑥ 時間軸: この結果は先月と比べてどう変わった？トレンドを把握しているか？
```

### Agent間のMMM壁打ち例
```
COMMANDER → ACQUISITION:
├── 「楽天の飽和度1.6%で投資余地大だが、+200%増額の実行リスクは検証したか？」
├── 「Meta Adsの-10%削減で、Shopify売上（相関0.67）への影響は試算したか？」
└── 「Google Adsの限界ROI=0は本当か？Hill R²を確認して信頼性を評価せよ」

COMMANDER → INSIGHT:
├── 「R²=0.71のモデルで細かい1%単位の予算配分を議論するのは過剰精度では？」
├── 「前回MMM結果との差分を出して、変化の要因を分析せよ」
└── 「外部要因のcoefficient符号がマイナスのものの原因を特定せよ」

COMMANDER → CFO:
├── 「限界CPA ¥15,000は目標LTVに対して許容範囲か？」
├── 「予算再配分による全体ROASへの影響を3シナリオ（Best/Base/Worst）で試算せよ」
└── 「プラットフォーム手数料を含めた真のチャネル別利益率を算出せよ」
```

## 連携プロトコル
- **ec-insight**: MMM実行・結果解釈・レポート作成の一次担当
- **ec-acquisition**: 飽和曲線・限界ROIに基づくチャネル別予算配分の実行
- **ec-commander**: 全チャネル横断の予算配分戦略判断。MMM結果レビューの最終承認
- **ec-operations**: プラットフォームチャネルのコスト構造・CVR連動分析
- **ec-creative**: 飽和原因がクリエイティブ疲弊の場合の改善施策連携
- **cfo**: ROI/CPA基準の財務的妥当性チェック。投資判断の最終承認
- **ec-engagement**: LTV観点でのCPA許容範囲設定。CRM施策との連動
