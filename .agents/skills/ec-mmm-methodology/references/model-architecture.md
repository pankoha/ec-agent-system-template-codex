# MMMモデル・アーキテクチャ詳細

## Robyn準拠Ridge回帰MMMの全体像

### 1. アドストック変換（Geometric Adstock）

広告の効果は出稿日だけでなく、翌日以降にも残存する。この「残存効果」をモデルに組み込む。

```
アドストック計算式:
adstocked[t] = spend[t] + decay_rate × adstocked[t-1]

例: Meta Ads（decay = 0.4）に日次¥100,000投下
  Day 1: ¥100,000（投下額そのまま）
  Day 2: ¥0 + 0.4 × ¥100,000 = ¥40,000（残存効果）
  Day 3: ¥0 + 0.4 × ¥40,000 = ¥16,000
  Day 4: ¥0 + 0.4 × ¥16,000 = ¥6,400
  → 4日間の累積効果: ¥162,400（投下額の1.62倍）
```

**チャネル別減衰率テーブル:**
| チャネル | 減衰率 | 半減期（日） | 効果残存期間（目安） |
|---------|--------|------------|-------------------|
| SEO | 0.70 | 1.9日 | 7-10日 |
| Google Ads | 0.50 | 1.0日 | 3-5日 |
| Meta Ads | 0.40 | 0.7日 | 2-4日 |
| Yahoo Ads | 0.35 | 0.6日 | 2-3日 |
| LINE | 0.25 | 0.4日 | 1-2日 |
| 楽天 | 0.20 | 0.3日 | 1-2日 |
| TikTok | 0.20 | 0.3日 | 1日 |
| Amazon | 0.15 | 0.2日 | 1日 |
| Email | 0.15 | 0.2日 | 1日 |

**減衰率の妥当性チェック:**
```
□ 認知型チャネル（Meta/TV）の減衰率が低すぎないか？（0.3-0.5が妥当）
□ 検索意図型（Amazon/Google Search）が高すぎないか？（0.1-0.3が妥当）
□ SEOの減衰率が他チャネルと同程度になっていないか？（0.6-0.8が妥当）
□ デフォルト値（0.3）が使われているチャネルは、実態に合っているか確認
```

### 2. Hill飽和曲線モデル

各チャネルの「投資 → 効果」の関係を非線形曲線でモデル化する。

```
Hill関数:
revenue = max_revenue × spend^S / (K^S + spend^S)

パラメータ:
├── max_revenue: そのチャネルから得られる理論上の最大日次売上
├── K（半飽和点）: max_revenueの50%に達する日次支出額
└── S（形状パラメータ）: 曲線の急峻度

例: K=¥50,000, S=1.5, max_revenue=¥500,000
  ¥10,000投下 → ¥37,000（飽和度7.4%、効率最高ゾーン）
  ¥50,000投下 → ¥250,000（飽和度50%、半飽和点）
  ¥100,000投下 → ¥396,000（飽和度79.2%、効率低下開始）
  ¥200,000投下 → ¥466,000（飽和度93.2%、追加効果わずか）
```

**フィッティング失敗時のフォールバック:**
```
Hill曲線のフィッティングが失敗する場合:
├── データポイントが少なすぎる（最低10日推奨）
├── 支出のバリエーションが少ない（毎日同額だとフィットできない）
└── 異常値（突発的に巨額を投下した日）が歪みを生む

フォールバック計算:
├── K = 中央値支出 × 2
├── S = 1.0（線形に近い仮定）
└── max_revenue = 最大日次売上 × 1.5
→ Hill R²が低い（0.1未満）場合、この飽和曲線は参考程度にとどめる
```

### 3. Ridge回帰（全チャネル統合モデル）

```
回帰式:
Y(t) = β₀ + Σ β_j × Hill(Adstock(spend_j(t))) + Σ γ_d × weekday_d(t) + Σ δ_k × context_k(t)

説明変数:
├── β_j: 各チャネルのHill変換済みアドストック支出
├── γ_d: 曜日ダミー変数（月〜土、日曜=基準）
└── δ_k: 外部要因（PR/TV/イベント/季節性）

Ridge正則化:
├── α = 1.0（デフォルト）
├── 多重共線性を抑制（チャネル間の相関が高い場合に重要）
└── 係数が過度に大きくなるのを防止

Ridge係数の解釈:
├── β_j > 0: そのチャネルへの投資は売上にプラス
├── β_j ≈ 0: そのチャネルの寄与は限定的
└── β_j < 0: データの問題（多重共線性など）の可能性。要注意
```

### 4. チャネルタイプの分類

```
広告チャネル（Advertising）:
├── 直接的な支出コントロールが可能
├── 予算最適化の対象
├── 例: Google Ads, Meta Ads, Amazon Sponsored, 楽天RPP, TikTok Ads

プラットフォームチャネル（Platform）:
├── 手数料ベース（売上の一定割合）で直接コントロール不可
├── 予算最適化の対象外（excluded=true）
├── 例: Shopify, BASE, MakeShop, WooCommerce, Yahoo Stores
├── ウィンドウ分析からも除外
└── ただし売上構成比・コスト構造の分析対象には含める
```

## モデル品質のトラブルシューティング

| 症状 | 原因候補 | 対処法 |
|------|---------|--------|
| R²が0.3未満 | データ期間不足、外部要因未登録 | 180日以上のデータを確保、セール日を登録 |
| 特定チャネルのβが負 | 多重共線性、データ品質 | チャネル間相関を確認、異常値を除外 |
| Hill R²が全チャネルで低い | 支出パターンが均一 | 意図的に予算変動を作る（テスト期間設定） |
| MAPEが50%超 | ノイズが大きい、外れ値 | データクレンジング、ロバスト推定を検討 |
| 飽和度が全チャネル100% | Kパラメータの推定失敗 | フォールバック値を確認、データ期間を延長 |
