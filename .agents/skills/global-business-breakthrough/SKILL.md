# Global Business Breakthrough — 毎朝配信スキル

## 概要
全世界の成功ビジネスを1日1社ピックアップし、インフルエンスダイヤグラムの思考法で構造分析。
メール（メルマガ形式）で配信し、Notionにアーカイブする。

## 配信先
- メール: your-email@example.com
- Notion: Global Business Breakthrough Library > ビジネス構造分析アーカイブ
  - 親ページID: YOUR_NOTION_PAGE_ID
  - DB data_source: collection://YOUR_NOTION_COLLECTION_ID

## 実行トリガー
- cronで毎朝7:57 JST に自動実行
- 手動実行: `/breakthrough` で即時実行も可能

## 企業選定ルール
- 過去にNotionアーカイブに登録済みの企業は重複しない
- 業界・地域をバランスよくローテーション
- 時価総額・規模の大小は問わない（スタートアップも大企業も対象）
- 日本企業も含む（ただし連続しない）

## 分析フレームワーク: インフルエンスダイヤグラム思考法

### インフルエンスダイヤグラムとは
ビジネスの成果（KPI）に影響を与える要因（ドライバー）の因果関係を図式化する手法。
「何が何に影響を与えているか」を構造的に明らかにすることで、
成功の本質的なメカニズムを理解する。

### 分析の手順
1. **最終成果（Output）の特定**: そのビジネスが達成した最も重要な成果は何か
2. **直接ドライバーの特定**: その成果に直接影響を与えた要因は何か（3〜5個）
3. **間接ドライバーの特定**: 直接ドライバーに影響を与えた背景要因は何か
4. **因果ループの発見**: ドライバー間の相互強化関係（フライホイール）はあるか
5. **ブレイクスルーの特定**: 上記構造の中で、最も非自明で決定的だった意思決定・発見・転換点は何か

### ブレイクスルーの定義
「それがなければ、このビジネスは今の形にならなかった」という決定的な転換点。
技術革新に限らず、以下のいずれかに該当するもの:
- ビジネスモデルの転換（ピボット）
- 非自明な意思決定（常識に反する選択）
- 市場の読み替え（既存市場の再定義）
- タイミングの妙（なぜ「今」だったのか）
- 構造的な参入障壁の構築方法

## メール配信フォーマット（HTML）

```
件名: [Global Business Breakthrough] {企業名} — {一行キャッチコピー}

本文構成:
1. ヘッダー（企業名・業界・地域・設立年・規模）
2. 一言サマリー（このビジネスを一文で表すと）
3. インフルエンスダイヤグラム（テキストベースの因果構造図）
   - 最終成果 ← 直接ドライバー ← 間接ドライバー
   - フライホイール（循環構造）があれば図示
4. ビジネスドライバー詳細（3〜5個）
   - 各ドライバーの説明と、なぜそれが効いたか
5. ブレイクスルー（最重要セクション）
   - 何が転換点だったか
   - なぜそれが非自明だったか
   - その判断の背景にあった洞察
6. EC支援事業・コンサル・D2C事業への応用ポイント（特定商品には言及しない。事業構造・戦略レベルの示唆）
7. フッター（Notionアーカイブへのリンク）
```

## Notion保存フォーマット

### ページ本文

## 実行手順

1. WebSearchで企業の最新情報を収集
2. インフルエンスダイヤグラム思考法で分析
3. メール本文をHTML形式で生成
4. Gmail MCP でドラフト作成 → 送信（gmail_create_draft）
5. Notion MCP でDBにレコード追加 + ページ本文を格納
6. 完了ログを出力
