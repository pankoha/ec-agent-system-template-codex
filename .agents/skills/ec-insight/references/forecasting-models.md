# 予測モデル 詳細

## 時系列予測モデル

### 入力変数
```
├── 過去売上（日次/週次）
├── 季節性指数
├── イベントカレンダー
├── 広告投下計画
├── 価格変更予定
├── 競合動向
└── 外部要因（景気、天候）
```

### 季節性指数
```
1月: 0.90（閑散）    7月: 1.05
2月: 0.85（最閑散）   8月: 1.00（お盆）
3月: 1.00（年度末）   9月: 1.15（楽天SALE）
4月: 0.95            10月: 1.10
5月: 1.00（GW）      11月: 1.30（BF）
6月: 1.10（楽天SALE） 12月: 1.50（年末）
```

### 出力
```
├── 点予測（中央値）
├── 80%予測区間
├── 95%予測区間
└── 予測精度スコア
```

## LTV予測モデル

### 計算式
```
基本LTV = 平均購入単価 × 購入頻度 × 継続期間
拡張LTV = Σ(将来購入確率 × 予想購入額 × 粗利率) / (1 + 割引率)^t
```

### セグメント別LTV
```
├── 新規顧客: ¥3,500
├── F2顧客: ¥8,000
├── F3+顧客: ¥15,000
├── ロイヤル: ¥35,000
└── VIP: ¥80,000
```

### LTV/CAC比率
```
├── 目標: 3.0以上
├── 良好: 2.0-3.0
├── 要改善: 1.0-2.0
└── 危険: 1.0未満
```

## 需要予測フレームワーク

### SKU別予測
```
├── Aランク: 週次予測、高精度
├── Bランク: 月次予測、中精度
└── Cランク: 四半期予測、低精度
```

### イベント補正
```
├── 楽天SALE: ×3-5倍
├── Amazon Prime Day: ×2-3倍
├── ブラックフライデー: ×2-4倍
├── 年末年始: ×1.5-2倍
└── 自社セール: ×2-3倍
```

### 安全在庫計算
```
安全在庫 = Z × σ × √リードタイム
├── Z=1.65: 95%サービスレベル
├── Z=2.33: 99%サービスレベル
└── σ: 需要の標準偏差
```
