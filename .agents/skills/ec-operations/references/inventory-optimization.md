# 在庫最適化 詳細

## 在庫管理パラメータ
```
発注点計算:
発注点 = 日販平均 × リードタイム + 安全在庫

安全在庫計算:
安全在庫 = Z × σ × √リードタイム
├── Z = 1.65: 95%サービスレベル
├── Z = 2.33: 99%サービスレベル
└── σ = 日販の標準偏差

経済発注量(EOQ):
EOQ = √(2 × 年間需要 × 発注コスト / 保管コスト)
```

## 倉庫配分
```
FBA:
├── 対象: Amazon販売商品、Prime必須
├── メリット: 配送速度、Buy Box優位
├── デメリット: 在庫上限、手数料
└── 配分: 売れ筋70%、その他30%

楽天倉庫:
├── 対象: 楽天主力商品、あす楽対象
├── メリット: 配送速度、楽天優遇
└── 配分: 売れ筋のみ

自社倉庫/3PL:
├── 対象: 自社EC、B2B、長期保管
├── メリット: 柔軟性、コスト
└── 配分: 残り全て、セット組み
```

## 配送最適化
```
送料設定: 送料無料ライン(AOV×1.2) / 一律¥500-800 / 地域別(離島のみ追加)
コスト削減: 複数キャリア比較 / サイズ最適化(60サイズ以下) / まとめ発送促進
配送品質: リードタイム2日以内 / 破損率0.1%以下 / 誤配送率0.05%以下 / 追跡率100%
```
