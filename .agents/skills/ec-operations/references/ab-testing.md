## テスト設計
```
├── 仮説: 「AをBに変更するとCVRがX%向上する」
├── 変数: 1つのみ変更
├── サンプルサイズ: 各バリアント最低1,000
├── 期間: 最低14日（週末2回含む）
├── 信頼度: 95%
└── 最小検出効果: 10%

統計的有意性:
├── p値 < 0.05: 有意
├── 信頼区間: 効果の範囲
└── 検出力: 80%以上
```

## カート離脱対策

### 離脱ポイント別
```
商品ページ → カート（30%離脱）:
├── 在庫残少表示
├── 「あと○○円で送料無料」
├── レビュー強調
└── 比較機能

カート → 会員登録（25%離脱）:
├── ゲスト購入オプション
├── SNSログイン
├── 入力項目削減（5項目以下）
├── 進捗バー表示
└── 離脱時ポップアップ

会員登録 → 決済（20%離脱）:
├── 決済方法追加（PayPay、楽天Pay等）
├── 分割払い表示
├── セキュリティバッジ
└── 返品保証表示

決済 → 完了（10%離脱）:
├── エラー時の明確なメッセージ
├── 再試行の簡易化
└── 別決済方法への誘導
```

### 離脱後リカバリー
```
1時間後: リマインドメール
24時間後: 在庫警告
72時間後: クーポン提供
目標回収率: 15%
```
