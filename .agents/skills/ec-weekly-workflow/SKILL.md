# 📅 週次最適化ワークフロー（22Agent版）

## ワークフロー概要
全7エージェント + PM + COMMANDERによる9ステップの週次最適化プロセス。
データ収集→分析→戦略調整→実行指示を毎週実行し、継続的な成長を実現する。

## 実行手順

### Step 1: INSIGHT（データ収集・統合）
```
├── データ収集・統合
├── KPI確認（売上/注文数/客単価/新規率）
├── チャネル別パフォーマンス集計
└── 異常検知アラートの確認
```

### Step 2: PROJECT MANAGER（進捗管理）
```
├── WBS進捗確認
├── リスク登録簿レビュー
├── 遅延タスクのリマインド
└── ルーティン実行確認
```

### Step 3: COMMANDER（戦略判断）
```
⚠️ 必須チェックリスト:
□ INSIGHTに売上・顧客データを確認したか？
□ ACQUISITIONに広告効率を確認したか？
□ CREATIVEにLP/クリエイティブ状況を確認したか？
□ ENGAGEMENTにCRM/リスト状況を確認したか？
□ OPERATIONSに在庫/CVR状況を確認したか？

├── 前週評価
├── 目標進捗確認
└── 優先順位決定
```

### Step 4: ACQUISITION（集客レビュー）
```
├── 広告パフォーマンス（ROAS/CPA/ACoS）
├── SEO状況（順位変動/流入）
├── チャネル別効率比較
└── 最適化提案（入札/クリエイティブ/KW）
```

### Step 5: CREATIVE（コンテンツレビュー）
```
├── コンテンツ効果（CVR/エンゲージメント）
├── SNS状況（フォロワー/ER/投稿効果）
├── クリエイティブ疲弊チェック
└── 制作タスク（今週の制作物）
```

### Step 6: OPERATIONS（運用レビュー）
```
├── 在庫チェック（アラート確認）
├── 価格レビュー（競合価格/自社価格）
├── CVR状況（サイト全体/商品別）
└── 配送・物流状況
```

### Step 7: ENGAGEMENT（顧客レビュー）
```
├── 顧客状況（RFMセグメント変化）
├── 配信効果（メール/LINE開封率・CVR）
├── CS状況（問い合わせ数・傾向）
└── リピート率・F2転換状況
```

### Step 8: PROJECT MANAGER（まとめ）
```
├── 議事録作成
├── Next Action整理（担当/期限明確化）
├── 請求・入金確認
└── 翌週タスク設定
```

### Step 9: COMMANDER（統合指示）
```
├── 全体統合（Step 1-8の情報統合）
├── 週次指示生成（各Agent向け）
└── リソース調整（必要に応じて）
```
