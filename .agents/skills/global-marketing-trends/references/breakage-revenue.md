# Breakage Revenue（未使用残高収益）詳細リファレンス

## 定義
Breakage Revenue = 購入された残高・ポイント・ギフトカードが使用されずに失効することで発生する収益

## グローバル市場規模

### 未使用額データ（年間）
| 企業/国 | 未使用額 | 備考 |
|---------|---------|------|
| **グローバル合計** | $21B（約3.1兆円） | 全ギフトカード・プリペイド |
| Amazon | $3B（約4,500億円） | 未換金負債として計上 |
| Starbucks | $1.4B（約2,100億円） | リワードポイント含む |
| Apple | $1.2B（約1,800億円） | App Store/iTunes |
| 米国全体 | $15.3B | 全プリペイドカード |

### Breakage率の実態
- **平均未使用率**: 20%（$100のギフトカードで$20が未使用）
- **完全未使用率**: 6%（一度も使われない）
- **部分未使用率**: 14%（残高が残る）

## 収益モデルの仕組み

```
┌─────────────────────────────────────────────────────────────────┐
│                    Breakage Revenue フロー                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  Step 1: 顧客がプリペイド残高を購入（¥10,000）                  │
│          ├── 企業: 現金受領、負債として計上                     │
│          └── 顧客: 将来の購入権利を取得                         │
│                                                                 │
│  Step 2: 時間経過（通常1-3年）                                  │
│          ├── 顧客が一部使用（¥8,000分購入）                     │
│          └── 残高¥2,000が残る                                   │
│                                                                 │
│  Step 3: 期限到来または放置                                     │
│          ├── 法的条件を満たす場合 → 収益認識                    │
│          └── 未使用¥2,000が純利益に                             │
│                                                                 │
│  【追加収益効果】                                               │
│  ├── 75%が額面以上を使用（アップセル効果）                      │
│  ├── 2.5倍の確率で定価購入（価格感度低下）                      │
│  └── 来店頻度向上（リピート効果）                               │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 日本の法規制

### 資金決済法の適用
| 項目 | 規制内容 |
|------|----------|
| **対象** | 前払式支払手段（プリペイドカード、電子マネー） |
| **発行保証金** | 未使用残高の50%を供託（1,000万円超の場合） |
| **有効期限** | 6ヶ月以上の有効期限が必要 |
| **届出義務** | 1,000万円超で届出、3億円超で登録 |

### 会計処理
```
【発行時】
  現金 10,000 / 前受金（負債） 10,000

【使用時】
  前受金 8,000 / 売上 8,000

【失効時（Breakage認識）】
  前受金 2,000 / 雑収入 2,000
```

## 成功事例詳細

### Starbucks Rewards
- **戦略**: 星（ポイント）に有効期限を設定（6ヶ月）
- **Breakage収益**: 年間$140M以上
- **心理効果**: 失効前の駆け込み購入を誘発
- **追加効果**: アプリ開封率向上、来店頻度UP

### Amazon ギフトカード
- **戦略**: 10年有効期限（法的最低限）
- **未換金残高**: $3B（貸借対照表に負債計上）
- **収益認識**: 統計的に使用されないと判断された時点で収益化
- **マーケティング**: 「贈り物」としてのポジショニング

### Costco メンバーシップ
- **戦略**: 年会費制（$60-$120/年）
- **更新率**: 92.6%（業界最高）
- **心理効果**: サンクコストで来店・購入を促進
- **収益構造**: 会員費が純利益の大部分を構成

## 日本EC向け応用設計

### 基本設計パターン

```
【パターン1: ポイント制度】
├── 付与率: 購入額の3-5%
├── 有効期限: 最終購入から1年
├── 失効通知: 30日前・7日前・前日
└── 期待Breakage率: 15-25%

【パターン2: プリペイド残高】
├── チャージ特典: +3-5%ボーナス
├── 有効期限: 2年間
├── 端数設計: 2,980円商品で220円残る設計
└── 期待Breakage率: 8-15%

【パターン3: 会員制ロスリーダー】
├── 年会費: ¥3,980-¥9,980
├── 特典: 送料無料、会員価格、先行販売
├── 更新率目標: 85%以上
└── LTV向上: 2-3倍
```

### 実装時の注意点
1. **法務確認**: 資金決済法の届出・登録要件
2. **会計処理**: 前受金の適切な管理
3. **顧客体験**: 失効通知の適切なタイミング
4. **ブランドリスク**: 「騙された」感を与えない設計

## KPI設計

| KPI | 計算式 | 目標値 |
|-----|--------|--------|
| Breakage率 | 失効額 / 発行額 | 15-20% |
| 有効利用率 | 使用額 / 発行額 | 80-85% |
| リチャージ率 | 再チャージ顧客 / 全チャージ顧客 | 40%+ |
| 上乗せ購入率 | 残高超過購入 / 全購入 | 60%+ |
| 平均上乗せ額 | (購入額 - 残高) / 上乗せ購入数 | ¥1,500+ |
