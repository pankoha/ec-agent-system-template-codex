# Platform-Specific Localization（日本ECプラットフォーム別ローカライズ）詳細リファレンス

## 定義
グローバルマーケティング施策を日本のECプラットフォーム（楽天、Amazon、Yahoo!、自社EC）の
特性に合わせて最適化する戦略。各プラットフォームの文化・機能・ユーザー層を理解した上での施策設計。

## 楽天市場攻略

### 楽天の特性
| 項目 | 特徴 | 施策への影響 |
|------|------|-------------|
| **ポイント経済圏** | SPU、マラソン、スーパーSALE | ポイント訴求必須 |
| **モール文化** | 出店者が店舗運営 | 店舗ブランディング重要 |
| **イベント依存** | 月2回の大型セール | イベント連動の売上構成 |
| **情報量重視** | 長文商品ページ | 詳細情報提供必須 |

### 楽天向け施策
```
【ポイント戦略】
├── SPU攻略: 全条件クリアで最大16倍
├── お買い物マラソン: 10店舗購入でポイント10倍
├── スーパーSALE: 半額商品、ポイントアップ
├── 5と0のつく日: +2倍（エントリー必須）
└── 楽天イーグルス勝利: +1倍

【商品ページ設計】
├── サムネイル: 情報量多め、価格・ポイント表示
├── 商品説明: 長文OK、スクロールで読む文化
├── 楽天内SEO: サジェストキーワード網羅
└── レビュー対策: フォローメールでレビュー依頼

【会員・リピート施策】
├── 店舗会員化: 次回クーポン配布
├── 楽天お気に入り: 値下げ通知活用
├── メルマガ: 楽天RMS経由の配信
└── 定期購入: 楽天定期購入便

【限定ドロップ応用】
├── スーパーSALE限定商品
├── マラソン連動セット
├── 店舗限定福袋
└── 先着限定クーポン
```

## Amazon Japan攻略

### Amazonの特性
| 項目 | 特徴 | 施策への影響 |
|------|------|-------------|
| **カート獲得** | Buy Box競争 | 価格・FBA重要 |
| **Prime中心** | 翌日配送が標準 | FBA必須級 |
| **アルゴリズム** | A9検索アルゴリズム | SEO対策必須 |
| **シンプルUI** | 情報は最小限 | 商品画像・タイトル勝負 |

### Amazon向け施策
```
【SEO・検索対策】
├── タイトル最適化: キーワード含有、読みやすさ
├── 箇条書き: 5点の特徴、キーワード埋込
├── A+コンテンツ: ブランドストーリー、比較表
├── バックエンドキーワード: 検索用キーワード設定
└── レビュー獲得: Amazon Vine、フォローメール

【広告・露出】
├── スポンサープロダクト: キーワード入札
├── スポンサーブランド: ブランド認知
├── スポンサーディスプレイ: リターゲティング
└── Amazon DSP: 外部配信

【Prime・定期便】
├── FBA活用: Prime対象化、カート獲得率UP
├── 定期おトク便: 15%OFF + まとめ割
├── Subscribe & Save: 自動配送設定
└── Prime限定セール: Prime会員向けディール

【限定ドロップ応用】
├── 数量限定タイムセール
├── Prime Day限定商品
├── ブラックフライデー限定
└── クーポン × 限定数量
```

### Amazon攻略ポイント
```
【カート獲得優先事項】
1. FBA利用（最重要）
2. 競争力のある価格
3. 在庫切れ防止
4. 出荷実績・評価
5. レビュー数・評価

【コンバージョン向上】
1. メイン画像の品質（白背景、高解像度）
2. サブ画像（使用シーン、サイズ感）
3. 動画コンテンツ
4. A+コンテンツの充実
5. レビュー数・質の向上
```

## Yahoo!ショッピング攻略

### Yahoo!向け施策
```
【PayPay・ポイント戦略】
├── 5のつく日: +4%PayPayポイント

├── ストアポイント: 店舗設定可能
├── PayPayクーポン: 店舗発行
└── プレミアム会員: +2%

├── 顧客セグメント: 購入回数、金額、商品

├── クーポン配布: 対象者限定
└── LINE連携: 友だち追加促進

【商品ページ設計】
├── 情報量: 楽天より少なめ、シンプル志向
├── 画像: 複数枚、使用イメージ
├── PR設定: PRオプションで露出UP
└── SEO: Yahoo!検索連動

【限定ドロップ応用】
├── 日曜日限定セット
├── PayPayジャンボ連動
├── 超PayPay祭り限定
```

## 自社EC攻略

### 自社ECの特性
| 項目 | 特徴 | 施策への影響 |
|------|------|-------------|
| **自由度** | 施策自由、データ取得 | 独自施策実行可能 |
| **顧客関係** | 直接コミュニケーション | CRM重要 |
| **利益率** | 手数料なし | 高利益率 |
| **集客課題** | 自力集客必須 | 広告・SEO投資必須 |

### 自社EC向け施策
```
【Breakage収益設計】
├── プリペイド残高: チャージ+3%ボーナス
├── ポイント制度: 購入額の3%、期限1年
├── 会員制: 年会費制プレミアム
└── サブスク: 月額/年額オプション

【ミステリーボックス】
├── お楽しみ袋: 定期的リリース
├── シーズン限定: 季節ごとの中身
├── 福袋2.0: 一部公開+サプライズ
└── サブスクボックス: 月替わり

【開封体験】
├── パッケージ投資: 価格帯に応じた設計
├── 同封物: 手書きメッセージ、ステッカー
├── サプライズ: 予告なしおまけ
└── ブランド統一: 世界観の演出

【限定ドロップ】
├── 会員限定先行販売
├── メルマガ限定商品
├── インスタライブ限定
└── アニバーサリー限定

【コミュニティ】
├── 会員ティア制度
├── アンバサダープログラム
├── レビュー特典
└── 紹介報酬制度
```

## クロスプラットフォーム戦略

### 統合運用
```
【在庫連動】
├── 中央在庫管理: 全チャネル一元管理
├── 安全在庫設定: チャネル別に設定
└── 自動振り分け: 売れ行きに応じて配分

【価格戦略】
├── 統一価格: 全チャネル同一（推奨）
├── モール別価格: 手数料に応じて調整
└── ポイント込み実質価格: 比較可能に

【施策連動】
├── 全チャネル同時セール
├── チャネル別限定商品
└── 自社EC優遇（高利益）
```

### チャネルミックス最適化
```
【売上構成目標例（Phase 4）】
├── Amazon: 42%（集客力活用）
├── 楽天: 27%（ポイント顧客）
├── Yahoo!: 8%（PayPayユーザー）
└── 自社EC: 23%（高利益、CRM）

【顧客獲得→自社EC誘導フロー】
1. モールで新規獲得
2. 同封物で自社EC告知
3. 自社EC限定特典で誘導
4. 自社ECでCRM、リピート化
```

## KPI設計

### プラットフォーム別KPI
| KPI | 楽天 | Amazon | Yahoo! | 自社EC |
|-----|------|--------|--------|--------|
| CVR | 3-5% | 8-15% | 2-4% | 2-4% |
| 客単価 | ¥5,000-8,000 | ¥3,000-5,000 | ¥4,000-6,000 | ¥6,000-10,000 |
| リピート率 | 20-30% | 15-25% | 15-25% | 30-50% |
| 手数料率 | 8-12% | 8-15% | 0-3% | 0% |

### 統合KPI
| KPI | 計算式 | 目標 |
|-----|--------|------|
| 全チャネル売上 | 各チャネル売上合計 | 月次+10% |
| 自社EC比率 | 自社EC売上 / 全体 | 20%→30% |
| 全体利益率 | (売上-コスト) / 売上 | 25%+ |
| クロス購入率 | 2チャネル以上購入者比率 | 10%+ |
