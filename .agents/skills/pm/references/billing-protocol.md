# 請求・キャッシュフロー管理プロトコル

## 発生主義（Accrual Basis）
```
売上と費用は、発生した期間に計上する。

❌ 幻の利益（Phantom Profit）
├── Month 1: 売上100万円（計上）
├── Month 6: 外注費80万円（計上）
└── → Month 1は利益100万に見えるが、実際は20万

✅ 正しい計上
├── Month 1: 売上100万、費用80万（発生主義）
└── → 利益20万が正確に把握できる
```

## 請求プロトコル
```
25日: 予算シート更新
├── 当月の作業内容と請求金額を確定

1日: 請求書発行 & 入金確認
├── 先月分の入金を確認
├── 当月分の請求書を発行

2日: 未入金フォローアップ
├── 入金がなければ即座にフォロー
├── 2回遅れたら法的措置も辞さない
```

## 請求チェックリスト
```
□ クライアント名は正確か
□ 請求金額は合意内容と一致しているか
□ 消費税の計算は正しいか
□ 振込先情報は正しいか
□ 支払期限は明記されているか
□ 作業明細は記載されているか
```

## データ・ファイル管理
```
Rule 1: Single Source of Truth
├── 全データはGoogle Driveのみ
├── ローカル保存禁止
└── バージョン管理はGoogle標準機能

Rule 2: Naming Convention（命名規則）
├── 形式: [YYMMDD]_[Client]_[Doc]_[Ver]
├── 例: 241012_ClientA_Proposal_v1.0
└── 例: 241020_ClientA_Invoice_v1.0

Rule 3: The "Old" Folder
├── 不要ファイルは即座に「00_Old」へ
└── 定期的に00_Oldを削除
```

## フォルダ構成テンプレート
```
📁 [プロジェクト名]
├── 📁 00_Old（旧ファイル置き場）
├── 📁 01_契約・見積
├── 📁 02_要件・設計
├── 📁 03_制作物
├── 📁 04_レポート
├── 📁 05_請求
└── 📁 99_参考資料
```
