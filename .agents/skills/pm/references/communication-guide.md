# コミュニケーション管理ガイド

## 内部コミュニケーション

### 敵: DM（ダイレクトメッセージ）
```
DMは情報の隠蔽。Truck Numberリスクの温床。

問題点:
├── 情報が属人化
├── 他メンバーが状況を把握できない
├── 退職時に情報が消失
└── 問題の早期発見ができない
```

### 目標: パブリックチャンネル90%以上
```
原則としてすべての業務連絡はパブリックチャンネルで。

例外（DMが許容されるケース）:
├── 人事評価
├── 給与交渉
├── 個人的な相談
└── センシティブな顧客情報
```

### 透明性を高める3つの施策
```
1. Soft Communication
├── 絵文字/スタンプを活用
├── 報告のハードルを下げる
└── 気軽に共有できる雰囲気

2. Public Correction
├── 指摘はパブリックで建設的に
├── チーム全体の学びになる
└── 個人攻撃はしない

3. 2-Steps Down
├── 自分の言動は2階層下まで影響する
├── リーダーの態度がチーム文化を作る
└── ジュニアは上の真似をする
```

## 外部コミュニケーション

### 3時間レスポンスルール
```
クライアントからの連絡には3時間以内に一次返信。

回答に時間がかかる場合でも:
「確認して本日中にご回答します」
と即座に返信する。
```

### 事前録画ルール（Pre-Video Technique）
```
【従来型（非効率）】
Meeting Start → Read Report (20m) → Discuss (10m) → End
└── 会議中に資料を読む時間が無駄

【桁違い式（高効率）】
Day Before: Send Video → Meeting Start → Discussion & Decision (30m) → End
└── 事前に動画で報告、会議は「決める時間」に集中

メリット:
├── クライアントは2倍速で視聴可能
├── 会議は議論と決定に集中
└── 顧客満足度が劇的に向上
```

### 会議設計テンプレート
```markdown
## 会議アジェンダ: [会議名]

### 事前準備（会議前日まで）
- [ ] 報告動画を送付
- [ ] アジェンダを共有
- [ ] 決定事項リストを準備

### アジェンダ
| 時間 | 内容 | 目的 | 担当 |
|------|------|------|------|
| 5min | オープニング | アイスブレイク | PM |
| 20min | 議論1 | 決定 | 全員 |
| 5min | クロージング | Next Action確認 | PM |

### 決定事項
1.
2.

### Next Action
| Action | 担当 | 期限 |
|--------|------|------|
```

### リマインドプロトコル
```
期限前日: 事前リマインド「明日が期限です」
期限当日: 確認「本日期限のタスクがあります」
期限後即時: 催促「期限を過ぎています。状況を教えてください」
期限後24h: エスカレーション「上長にCC」
```
