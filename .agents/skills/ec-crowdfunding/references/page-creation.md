# ページ制作 詳細ガイド

## 最重要: CVRがすべてを決める

> **CVR 1%改善 → 売上100万円の差**
> クラファンでは1万回以上ページが閲覧される。ページのクオリティが売上を最も左右する。

## コンセプト設計フレームワーク

### POP/POD/POF分析
```
POP分析（Point of Parity: 同等要素）:
├── 市場で必須の要素を把握する
├── 「これがないと検討すらされない」項目
└── 例: 枕なら「洗える」「高さ調整」「通気性」

POD分析（Point of Difference: 差別化要素）:
├── 自社の強み・USPを発見する
├── 「これがあるから選ばれる」項目
└── 例: 「世界初の○○素材」「特許取得技術」

POF分析（Point of Failure: 失敗要素）:
├── 避けるべき要素を把握する
├── 「最悪シナリオ」を事前に想定して排除
└── 例: 「すぐ壊れる」「サイズが合わない」「充電が持たない」
```

### 競合分析 3つの軸
| 競合タイプ | 説明 | 例 |
|---|---|---|
| **Time競合** | 時間の奪い合い | Netflix vs 外食 |
| **Job競合** | 同じ課題を解決する代替品 | 電気毛布 vs エアコン |
| **Category競合** | 同カテゴリの直接競合 | A社の枕 vs B社の枕 |

### コンセプトコピー4類型
```
① 素材新規性型: 「世界初の○○素材を採用」
   └── 技術・素材の新しさで差別化

② 課題解決型: 「○○に悩むあなたへ」
   └── ターゲットの具体的な悩みを解決

③ 感情/価値訴求型: 「自分へのご褒美」「至福の体験」
   └── 感情・ライフスタイル的な価値を訴求

④ 価格訴求型: 「メーカー直販だから実現できた価格」
   └── コストパフォーマンスの高さで訴求

★ コンセプトを量産 → 広告テストで検証 → 絞り込み → KV画像作成 → タイトル作成
```

### リサーチの実践手順
```
Step 1: 市場ボリューム確認
├── Googleキーワードプランナーで検索量を確認
└── 十分な需要（月間検索1,000+）があるか判断

Step 2: 商品の提供価値整理
├── XMindでマインドマップ作成
└── 全特徴を洗い出し、分類する

Step 3: 競合リサーチ
├── 海外ECの口コミ（自社EC以外も必ず確認）
├── Amazon・楽天の少数購入者の口コミ（★1-3が宝の山）
├── Googleの検索結果上位3位（= 消費者の常識を形成）
├── Googleサジェストでリアルな検索キーワードを把握
└── マイベスト（https://my-best.com/）で比較軸を理解

Step 4: POP/POD/POF抽出
└── リサーチ結果から各要素を整理

Step 5: コンセプト決定 → 広告テストで検証
```

## 売れる構成の型

### 型①: 問題提起型（課題解決商品向け）
```
FV → 悩み共感 → 解決策提示 → 機能説明 → エビデンス → CTA

1. ヘッダーKV（最重要）
   ├── 商品画像 + キャッチコピー（コンセプト4類型から選択）
   ├── 占有率85%以上、白背景推奨
   └── ★ コピーを変えるだけで成果が大幅改善/悪化する

2. 問題提起セクション
   ├── 「こんなお悩みありませんか？」
   └── 3-5個の具体的な課題を提示

3. 解決策の提示
   └── ビフォー/アフターの明示

4. 特徴・機能詳細
   ├── POP分析で抽出した項目を全て入れる
   ├── PODを強調して差別化
   ├── 比較表（従来品 vs 本商品）
   └── エビデンス（試験結果・認証）

5. 使用シーン
   └── ライフスタイル画像3-5枚

6. スペック・仕様
   └── 何が届くのか視覚的に伝える

7. リターン紹介
8. FAQ
9. プロジェクトオーナー紹介
```

### 型②: 機能訴求型（スペック重視商品向け）
```
FV → 権威性 → 機能詳細 → 比較表 → 利用シーン → CTA

1. ヘッダーKV
   ├── 商品の全体像 + スペックハイライト
   └── 「業界初」「特許取得」等の権威付け

2. コア機能紹介（3つ以内に絞る）
   └── 最大の差別化ポイントを最初に + 数値データ

3. 技術解説
   └── 第三者機関のテスト結果

4. 実使用デモ
   └── GIF/動画で動作を見せる

5-9. （型①と同様）
```

## 構成要素チェックリスト
```
☑ POP分析で抽出した項目を全て入れる
☑ PODを明確に強調している
☑ 何が届くのか視覚的に伝える（同梱物一覧画像）
☑ エビデンス（試験結果・認証）を取得・掲載している
☑ 権威性・第三者の声を入れている
☑ GIFアニメーション活用（Makuake: 7MB以内、CAMPFIRE: 4MB以内）
☑ モバイルでも読みやすいレイアウト（14px以上）
☑ 利用シーンのライフスタイルショットがある
☑ 法的表記（薬機法・景表法）に問題がない
```

## エビデンス取得先
```
第三者試験機関:
├── eurofins（ユーロフィン）: 国際的な検査機関
├── カケンテストセンター: 繊維・日用品の試験
├── ボーケン品質評価機構: 耐久性・安全性試験
├── SGS: グローバル検査機関
└── 費用: 数万円〜（試験項目による）

効果:
├── 「第三者機関による試験で○○を実証」は強力なエビデンス
├── CVRを大幅向上させる説得力
└── 広告クリエイティブでも使える
```

## 外注先ガイド
```
デザイナー・クリエイター:
├── ランサーズ: フリーランスマッチング
├── クラウドワークス: フリーランスマッチング
├── ココナラ: スキルマーケット
└── Talema（タレマ）: クリエイター紹介

デザイン指示:
├── AUN（ビジュアルコミュニケーションツール）を活用
├── URLのスクリーンショットを共有して指示
└── 修正指示も視覚的にコミュニケーション

動画制作:
├── 15-30秒の商品紹介動画
├── GIFアニメーション（使用感を伝える）
└── 字幕必須（音声OFFでも伝わる）
```

## コピーライティングフレームワーク

### PASONA法則（CF向けアレンジ）
```
P（Problem）: 問題提起 → ターゲットの悩みを描写
A（Affinity）: 親近感 → 開発者も同じ悩みを持っていた
S（Solution）: 解決策 → 商品が解決策
O（Offer）: 提案 → Makuake限定35%OFF
N（Narrowing）: 絞り込み → 限定100台
A（Action）: 行動 → 今すぐ応援購入
```

## Custom GPTs活用（AI支援）
```
GPTsで自動化できる領域:
├── LP本文の生成（商品特徴入力→構成案出力）
├── 広告コピーの量産（コンセプト→複数パターン生成）
├── LINE配信文のカスタマイズ
├── FAQ生成（商品仕様→想定質問と回答）
└── ★ AI生成コンテンツは必ず人間がレビュー・修正すること
```
