# LINE集客戦略 詳細ガイド

## 核心原則

> **初日売上の公式: LINE友達数 × 購入率 × 客単価**
> LINE集客がCFの成否を決定する。5日間配信が最新ベスト。

## LINE 5日間配信プロトコル（最新版）

> **7日間配信は古い。4-5日目から開封率が低下するため、5日間が最適。**

### 4つの壁を突破する設計
```
Day 1 「認知の壁」:
├── テーマ: プロジェクト・商品の存在を知る
├── 件名: 「○○プロジェクト、まもなくスタート！」
├── 内容:
│   ├── プロジェクト概要（PODを1つだけ）
│   ├── 「LINE限定の先行割引があります」の予告
│   └── 次回配信の予告
└── 目的: 「こんな商品があるんだ」と認知させる

Day 2-3 「興味の壁」:
├── テーマ: もっと知りたいと思わせる
├── 件名: 「開発ストーリー公開！なぜこの商品を作ったのか」
├── 内容:
│   ├── 開発者の想い・ストーリー（プロセスエコノミー）
│   ├── 商品の特徴詳細（2-3点）
│   ├── 使用シーンの紹介（ライフスタイルショット）
│   └── エビデンス（第三者試験結果）
└── 目的: 「この商品、いいかも」と興味を持たせる

Day 3-4 「信頼の壁」:
├── テーマ: このプロジェクトは信頼できると思わせる
├── 件名: 「【限定公開】リターン内容と割引率を先行公開！」
├── 内容:
│   ├── リターン構成の詳細（4段階）
│   ├── 各割引率の説明
│   ├── 「超早割は○個限定」の希少性訴求
│   ├── 「開始日時: ○月○日 ○時」の告知
│   └── Googleカレンダー登録リンク（忘れ防止）
└── 目的: 「買おうかな」と購入意欲を持たせる

Day 5 「行動の壁」（当日）:
├── テーマ: 今すぐ行動させる
├── ★ 3回配信が必須（1通だけは絶対NG）
│
├── 12:00（公開直後）:
│   ├── 件名: 「【START】ついに公開！超早割はこちら→」
│   ├── プロジェクトページへの直接リンク
│   ├── スタートダッシュクーポン添付
│   └── リッチメッセージ使用（CTR大幅向上）
│
├── 14:00（2回目）:
│   ├── 件名: 「【速報】○時間で○万円達成！」
│   ├── 実績報告 → ソーシャルプルーフ（FOMO喚起）
│   ├── 超早割の残数報告
│   └── 「まだ間に合います」
│
└── 18:00（3回目）:
    ├── 件名: 「超早割 残り○個…本日限り」
    ├── 残枠の緊急性訴求
    ├── SNSシェア依頼（類似属性リーチ）
    └── 「応援購入した方はぜひシェアを」
```

### 前日配信（Day 4）
```
Day 4（前日）:
├── 件名: 「【明日開始】超早割は争奪戦の可能性が…」
├── 内容:
│   ├── 「いよいよ明日○時スタート！」
│   ├── 超早割の限定数リマインド
│   ├── プロジェクトページのリンク（事前ブックマーク推奨）
│   ├── Googleカレンダー登録の再案内
│   └── 「開始と同時にアクセスしてください」
└── 目的: 「明日絶対にチェックする」と決意させる
```

## LINE公式アカウント構築

### 初期設定
```
Step 1: アカウント作成
├── プラン: フリープラン → 従量課金プランへ移行検討
├── プロフィール: ブランドロゴ・説明文
└── あいさつメッセージの設定

Step 2: リッチメニュー設定
├── 構成例:
│   ├── 「プロジェクトを見る」→ CFページリンク
│   ├── 「特典を確認」→ リターン一覧
│   ├── 「お問い合わせ」→ チャット or FAQ
│   └── 「SNSフォロー」→ Instagram
└── サイズ: 大（2500x1686px）推奨

Step 3: 友だち追加導線の構築
├── PC登録: QRコード + URL
├── スマホ登録: ディープリンク（line://）
├── ティザーサイト: CTAボタン設置
├── Meta広告: LP → LINE追加の導線
└── SNS: プロフィールリンク・投稿内CTA
```

### 友だち追加インセンティブ
```
効果的な訴求:
├── 「LINE限定 最大35%OFF 先行割引」
├── 「LINE友だち限定 超早割のご案内」
├── 「登録者だけに開始日時を先行告知」
├── 「LINE限定 スタートダッシュクーポン」
└── ★ 割引率の数字を明示するとCVR向上
```

## リッチメッセージ活用

> **リッチメッセージはテキストメッセージよりCTRが大幅に向上する。必ず使う。**

```
テンプレート:
├── ティザー期: 商品ビジュアル + 「Coming Soon」+ CTA
├── ローンチ当日: 商品画像 + 「NOW ON SALE」+ 「超早割35%OFF」+ CTA
├── 中間報告: 達成率グラフ + 残リターン案内 + CTA
└── サイズ: 1040x1040px推奨

★ テキストだけの配信は開封されても行動されにくい
★ リッチメッセージは視覚的訴求力でクリック率が格段に高い
```

## Googleカレンダー連携（忘れ防止）
```
設定方法:
├── 「○月○日○時 ○○プロジェクト開始」のカレンダーイベントリンクを作成
├── LINE配信にカレンダー登録リンクを添付
├── Day 3-4の配信で案内
└── ★ 「登録しておけば通知が届きます」と案内

効果:
├── 公開日にスマホ通知が届く → 忘れ防止
├── 友だち追加したが配信を見逃した人にもリーチ
└── 別チャネルでのリマインド効果
```

## SNSシェア依頼
```
タイミング: 当日18時の3回目配信で依頼

内容:
├── 「応援購入いただいた方、ぜひSNSでシェアをお願いします！」
├── シェアしやすいリンク・テキストを用意
├── ★ 支援者と類似属性の友人にリーチできる
└── ★ 口コミ効果で新規支援者を獲得

ポイント:
├── 共有ボタンをLINE配信に設置
├── ハッシュタグを指定
└── シェアした人への特典があると効果UP
```

## KV/タイトルA/Bテスト
```
テスト方法:
├── Meta広告で複数クリエイティブを同時テスト
├── KV画像のバリエーション × コンセプトコピー4類型
├── 勝ちパターンをCFプロジェクトページに反映
└── ★ 広告テスト段階で最もCTR/CVRの高い組み合わせを発見

判断基準:
├── CTR > 1-2%のクリエイティブが「勝ち」
├── CPA ¥600以下を達成できるものを採用
└── 最低50CV以上のデータで判断
```

## 「もうすぐ開始」機能の活用
```
Makuake審査通過後に利用可能:
├── プロジェクト公開前に「もうすぐ開始」ページが生成
├── ユーザーが「通知を受け取る」で事前登録
├── 公開日にMakuakeから自動通知
└── ★ Meta広告のLPとしても使用可能（LINE登録と併用）
```

## 配信タイミング最適化
```
CF特化の推奨:
├── ローンチ当日: 12時(公開) → 14時(実績) → 18時(残枠)
├── 通常配信: 火〜木の12時 or 19時
├── 緊急告知: 即時配信OK
└── ★ 当日は必ず3回配信（1通だけは厳禁）

開封率8割の重要性:
├── LINE配信の開封率は通常60-80%
├── 8割まで開封されるまで売上が伸びる
├── 開封率が低い → 配信タイミングの見直し
└── ★ 開封率を注視し、最適な時間帯を見つける
```

## LINE友だち数と売上の相関
```
目安（転換率10%、平均支援額¥10,000の場合）:
├── 500友だち → 50件 → 50万円
├── 1,000友だち → 100件 → 100万円
├── 3,000友だち → 300件 → 300万円（ランキング4位以内の参考）
├── 5,000友だち → 500件 → 500万円
└── 10,000友だち → 1,000件 → 1,000万円

★ 初日売上150万円+ → ランキング4位以内 → 最終売上3倍
★ 友だち1,500人 × 転換率10% × 単価¥10,000 = 初日150万円
```
