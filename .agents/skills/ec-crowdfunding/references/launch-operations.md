# ローンチオペレーション 詳細ガイド

## 核心: 初日150万円+ → ランキング4位以内 → 最終売上3倍

> **最終到達金額 ≒ 初日売上 × 3倍**（統計的データ）
> スタートダッシュがその後のすべてを決める。

## 初日の目標設定
```
ランキング4位以内のベンチマーク:
├── 初日売上 ¥150万円以上 → ランキング4位以内の目安
├── ★ 前日にMakuakeの現在のランキング状況を確認すること
├── ランキング上位 → ピックアップ掲載 → メルマガ掲載 → 2-3万PV獲得
└── 最終到達金額 ≒ 初日 × 3倍

達成に必要なLINE友だち数の逆算:
├── 初日目標¥150万円 ÷ 平均単価¥10,000 = 150件
├── 150件 ÷ 転換率10% = 1,500友だち必要
└── ★ 広告予算 = 1,500人 × CPA¥600 = ¥900,000
```

## 公開日・時刻の決定
```
★ 火曜日 11時公開がベスト

理由:
├── 売れている状態で土日を迎えられる
├── メルマガの新着・カテゴリ枠に掲載されやすい
├── 11時 → スタートダッシュクーポンの有利性
├── 午前公開 → 夕方のメルマガに掲載される可能性
└── 「もうすぐ開始」機能で事前通知登録者も公開時に通知

避けるべき:
├── 土日 → メルマガ枠が週間ランキング/ラストチャンスのみ
├── ※ ただし事前集客で十分なリストがあれば土曜もOK
└── 金曜 → 週末前で購買が分散
```

## ローンチ当日タイムライン（最重要）

### 11:00 プロジェクト公開
```
├── Step 1: プロジェクト公開を確認（1分以内）
├── Step 2: ページ表示・リターンの正常動作確認
├── Step 3: ティザーサイトのリンク差し替え（CFページへ）
└── Step 4: 広告の配信開始設定を確認
```

### 12:00 LINE配信①（公開報告）
```
★ 公開直後の第1回配信（最重要）

配信内容:
├── 「ただいまスタートしました！」
├── プロジェクトページへの直接リンク
├── 超早割リターンへのディープリンク
├── スタートダッシュクーポン添付
├── リッチメッセージ形式で配信（CTR大幅向上）
└── 「限定○個、お早めに！」

★ この配信で初日売上の50-70%が決まる
```

### 14:00 LINE配信②（実績FOMO）
```
★ 2回目配信で「乗り遅れたくない」心理を刺激

配信内容:
├── 「【速報】○時間で○万円達成！」
├── 超早割の残数報告（「残り○個」）
├── 達成率のビジュアル表示
├── 「まだ間に合います！」
└── ★ 1回目で行動しなかった人の背中を押す

効果:
├── ソーシャルプルーフ（他の人も買っている）
├── FOMO（Fear of Missing Out: 乗り遅れ恐怖）
└── 限定数の減少 → 緊急性の強化
```

### 18:00 LINE配信③（残枠・緊急性）
```
★ 3回目配信でゴールデンタイムの購入を促進

配信内容:
├── 「超早割 残り○個…」
├── 本日の達成状況の総括
├── SNSシェア依頼（類似属性リーチ）
├── 「応援購入した方はぜひお友達にもシェアを」
└── ★ 夕方以降はEC購買のゴールデンタイム

★ 3回配信で開封率8割まで到達すれば成功
```

### 複数回配信の重要性
```
❌ 「公開しました！」の1通だけ → 絶対NG（売上の半分以上を失う）
✅ 3回配信（12時/14時/18時）→ 開封率8割まで押し上げる

なぜ3回必要か:
├── 12時: 全員に告知（開封率30-40%）
├── 14時: 12時に見逃した人 + 実績でFOMO喚起（累計開封率50-60%）
├── 18時: まだ見ていない人 + 残枠の緊急性（累計開封率70-80%）
└── ★ 開封率8割まで売上が伸び続ける。1通だけでは8割到達しない

配慮事項:
├── 「迷惑ではないか」と心配する必要はない
├── CFは期間限定の特別なイベント → 複数配信は当然
└── 支援者は情報を求めている → 送らないほうが機会損失
```

│   ├── 「LINE友だち○○人を獲得済み」
│   ├── 「初日○万円の売上見込み」
│   └── 「Meta広告のCPA ¥○○で○○人集客済み」
├── メルマガ掲載のリクエスト
│   ├── 新着メルマガへの掲載
│   ├── カテゴリ別メルマガへの掲載
│   └── 週間ランキングメルマガ
└── ★ 数値で示すことで掲載の優先度が上がる
```

```

Step 5: 配送実行 → 追跡番号発行 → 発送通知
