# リターン設計 詳細ガイド

## 4段階リターン構成テンプレート

### 基本構成（Makuake推奨）
```
リターン構成（単品商品の場合）:

Tier 1: 超早割（Super Early Bird）
├── 割引率: 30-40% OFF
├── 数量: 極少数（LINE友だちの30-50%が目安）
├── 目的: スタートダッシュ・ランキング獲得
├── 対象: LINE事前登録者優先
├── 表示例: 「【超早割 35%OFF】○○ 1個 ¥X,XXX」
└── ★ 即完売がソーシャルプルーフになる

Tier 2: 早割（Early Bird）
├── 割引率: 20-30% OFF
├── 数量: 200個程度
├── 目的: 初速の継続
└── 表示例: 「【早割 25%OFF】○○ 1個 ¥X,XXX」

Tier 3: Makuake割（プラットフォーム割）
├── 割引率: 10-20% OFF
├── 数量: 300-500個
├── 目的: 中間期の安定売上
└── 表示例: 「【Makuake割 15%OFF】○○ 1個 ¥X,XXX」

Tier 4: セット販売（AOV向上）
├── 2個セット: カップル向け（1個あたり20-25%OFF）
├── 3個セット: ファミリー向け（1個あたり25-30%OFF）
├── 目的: 客単価向上（AOV最大化）
└── ★ セット販売で平均注文額が大幅に向上する
```

### リターン画像の必須要素
```
ライフスタイルショットが必須:
├── 商品を使っているシーンの写真
├── 白背景の商品写真だけではダメ
├── ターゲットが「自分ごと」として想像できるシーン
├── 使用環境（リビング/オフィス/アウトドア等）で撮影
└── ★ 「こんな風に使える」が伝わると購入率UP

リターン画像チェック:
□ 各リターンの内容が一目で分かる
□ 割引率・お得感が明確に表示
□ 限定数が記載されている
□ 配送予定時期が記載されている
□ セット内容が視覚的に分かる（複数商品の場合）
□ サイズ感が伝わる参考画像あり
□ ライフスタイルショットが含まれている
```

## 割引率設計の原則

### 利益確保の計算ロジック
```
例: 一般販売価格 ¥10,000 の商品（超早割35%OFF）

販売価格: ¥6,500
├── 手数料(20%): -¥1,300
├── 原価(製造+送料+関税): -¥2,000
├── 国内配送: -¥800
├── 広告費/個: -¥600
├── 総コスト: -¥4,700
├── 利益: ¥1,800
└── 利益率: 27.7%

★ 超早割でも最低20%の粗利確保が必須
```

### 割引率の上限・下限
```
├── 最大割引率: 35-40%（これ以上は利益困難）
├── 最小割引率: 10-15%（これ未満はCFのメリット薄い）
├── Tier間の差: 5-10%ポイント
│   ├── 差が小さすぎる → 超早割のインセンティブ不足
│   └── 差が大きすぎる → 後半リターンの魅力不足
└── 利益率の目安:
    ├── 超早割: 最低20%の粗利
    ├── 早割: 25-30%の粗利
    ├── 通常割: 30-35%の粗利
    └── セット割: 25-35%の粗利
```

## 数量設計の考え方

### 需要予測に基づく数量設定
```
Step 1: 目標支援金額の設定（例: 目標500万円）
Step 2: 平均支援単価の想定（例: ¥8,000）
Step 3: 必要支援件数の計算（625件）
Step 4: Tier別配分
├── 超早割: 15-20% → 100個
├── 早割: 25-30% → 175個
├── 通常割: 35-40% → 250個
├── 2個セット: 10-15% → 75セット
└── 3個セット: 5-10% → 25セット
```

### 品切れ戦略（SOLD OUT活用）
```
早期Tier（超早割・早割）の数量は意図的に少なく設定:
├── 「SOLD OUT」表示がソーシャルプルーフになる
├── 「急がないと次のTierになってしまう」という緊急性
├── ランキング上位表示の効果で次のTier支援が加速
└── ★ LINE友だち数の30-50%程度を超早割数量の目安に
```

## アクティビティレポートで信頼構築
```
投稿するコンテンツ:
├── 達成報告: 「○%達成！ありがとうございます」
├── 開発進捗: 「製造工程の様子を公開」
├── 使用レビュー: 「チームメンバーが実際に使ってみた」
├── ストレッチゴール: 「次の目標と追加特典を発表」
└── メディア掲載: 「○○メディアに紹介されました」

効果:
├── 支援者とのコミュニケーション → 追加支援・SNSシェア促進
├── 信頼性構築 → 未支援者の不安解消 → CVR向上
├── プロジェクトページのSEO効果
└── ★ 定期的なレポートが「このプロジェクトは信頼できる」の証明になる
```

## 配送タイムラインの設定
```
適切な設定方法:
├── 製造期間 + 輸送期間 + 検品期間 + 配送期間 + バッファ
├── バッファ: 全工程の+20-30%
├── 例: 製造2ヶ月 + 輸送1ヶ月 + 検品2週間 + 配送1週間 + バッファ1ヶ月
│   └── 合計: 約5ヶ月後の配送予定
├── ★ 楽観的すぎる配送時期は信頼を損なう
└── ★ 少し余裕を持った時期設定 → 早く届けば好印象

配送遅延時の対応:
├── 活動レポートで事前に理由を説明
├── 新しい配送予定日を明確に提示
└── 丁寧なコミュニケーションが最重要
```

## ストレッチゴール設計
```
段階的目標設定:
├── 初期目標: ¥1,000,000 → 「目標達成！」
├── ストレッチ1: ¥2,000,000 → カラバリ追加
├── ストレッチ2: ¥3,000,000 → 限定アクセサリー同梱
├── ストレッチ3: ¥5,000,000 → 全支援者アップグレード
└── ストレッチ4: ¥10,000,000 → 限定モデル制作決定

設計ポイント:
├── 各ゴール間は達成可能な距離に
├── 特典は原価が低く、満足度が高いものを選ぶ
├── 中間期の中だるみ対策として段階的に発表
└── SNSで共有しやすいビジュアル報告を準備
```

## 広告シミュレーション表
```
入力項目:
├── 一般販売予定価格
├── 原価（製造+送料+関税）
├── 国内配送料
├── 手数料率（20%）
├── 各Tierの割引率・数量
├── 広告予算（総額）
├── 想定CPA（LINE友だち追加: ¥600目標）
├── LINE→支援 転換率（10%目安）

出力項目:
├── 各Tierの売上見込み
├── 総支援金額見込み
├── 広告費率（目標20%以下）
├── 粗利額・粗利率
├── ROI・損益分岐点
```
