# 戦略策定前 必須データ収集チェックリスト

## 戦略策定前チェックリスト（MANDATORY）
```
┌─────────────────────────────────────────────────────────────────┐
│  □ Step 1: INSIGHTに売上・顧客データを確認したか？              │
│  □ Step 2: ACQUISITIONに広告効率を確認したか？                  │
│  □ Step 3: CREATIVEにLP/クリエイティブ状況を確認したか？        │
│  □ Step 4: ENGAGEMENTにCRM/リスト状況を確認したか？             │
│  □ Step 5: OPERATIONSに在庫/CVR状況を確認したか？               │
│                                                                 │
│  上記すべてにチェックが入るまで、戦略決定に進んではならない      │
└─────────────────────────────────────────────────────────────────┘
```

## 対象コマンド（このチェックリストが必須）
- `/commander` - 戦略・フェーズ・競合
- `/phase` - フェーズ判定
- `/weekly` - 週次最適化（COMMANDERステップ）
- `/launch` - 新商品ローンチ
- 戦略に関する自由形式の質問

## 必須データ収集フロー詳細
```
Step 1: INSIGHT（データ収集）
├── 売上データ（チャネル別、月別、商品別）
├── 広告データ（ROAS、CPA、ACoS）
├── 顧客データ（LTV、リピート率、F2転換率）
└── 予測データ（需要予測、季節性）

Step 2: ACQUISITION（集客状況）
├── チャネル別パフォーマンス
├── 広告効率の推移
├── SEO順位・流入状況
└── 競合の広告動向

Step 3: CREATIVE（コンテンツ状況）
├── LP/クリエイティブのCVR推移
├── クリエイティブ疲弊状況
└── SNSエンゲージメント

Step 4: ENGAGEMENT（顧客状況）
├── CRMデータ（リスト数、開封率、CVR）
├── RFM分析結果
├── CS問い合わせ傾向
└── NPS/顧客満足度

Step 5: OPERATIONS（運用状況）
├── 在庫状況・回転率
├── CVR推移（サイト全体）
├── 価格競争力
└── 物流キャパシティ
```

## プロセス違反チェック
```
❌ 以下の場合、戦略策定を中止し、データ収集からやり直すこと：

□ INSIGHTからの売上・顧客データなしに戦略を策定しようとした
□ ACQUISITIONからの広告効率データなしにマーケ予算を決めようとした
□ OPERATIONSからの在庫データなしに販売計画を立てようとした
□ ENGAGEMENTからのCRMデータなしにLTV施策を決めようとした
□ 単独の推測・仮定のみで戦略を策定しようとした
```
